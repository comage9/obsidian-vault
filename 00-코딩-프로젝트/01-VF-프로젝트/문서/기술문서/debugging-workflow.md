# 디버깅 워크플로우 (5단계)

> Source: agent-skills (Addy Osmani) 참고

## 5단계 프로세스

### 1. 재현 (Reproduce)
- 에러를 확실하게 재현
- 최소 조건 파악: 어떤 입력, 어떤 환경에서 발생?
- 재현 불가면 로그/모니터링 추가 후 대기

### 2. 국소화 (Localize)
- 문제 영역 좁히기
- 이진 탐색: 절반씩 제외하며 원인 위치 특정
- 로그/print/curl로 범위 축소

### 3. 축소 (Reduce)
- 최소 재현 케이스 만들기
- 관련 없는 변수 제거
- 핵심 조건만 남기기

### 4. 수정 (Fix)
- 근본 원인 해결 (증상이 아닌 원인을 고칠 것)
- 파편화된 수정 금지 (한 곳에서 깔끔하게)
- 수정 후 전체 동작 재확인

### 5. 방지 (Guard)
- 재발 방지 테스트 작성
- RULES/code.md에 규칙 추가 (필요시)
- wiki/concepts/에러로그.md에 기록

## Stop-the-line 규칙
- 같은 에러 2회 발생 → 즉시 작업 중단 → 근본 원인 분석
- 3회 → 접근 방식 자체를 재고
