# 옵시디언 검색 시스템 구성 문서

## 📋 시스템 개요

옵시디언 문서를 PostgreSQL 데이터베이스에 저장하고 웹 인터페이스로 검색할 수 있는 시스템입니다. GBrain의 WebAssembly 문제를 우회하여 안정적인 검색 시스템을 구축했습니다.

## 🏗️ 시스템 아키텍처

```
옵시디언 파일 (*.md)
    ↓ (자동 동기화, 5분 간격)
PostgreSQL 데이터베이스 (Docker 컨테이너)
    ↓ (REST API)
웹 검색 인터페이스 (Flask)
    ↓ (사용자)
웹 브라우저 / API 클라이언트
```

## 🔧 구성 요소

### 1. PostgreSQL 데이터베이스
- **컨테이너**: `gbrain-postgres`
- **포트**: 5432
- **데이터베이스**: `gbrain`
- **사용자**: `gbrain_user`
- **비밀번호**: `gbrain_password_123`
- **데이터 볼륨**: `~/gbrain-docker/postgres_data`

### 2. 웹 검색 인터페이스
- **포트**: 5000
- **기술**: Flask + PostgreSQL
- **기능**: 문서 검색, 통계, 태그 필터링
- **주소**: http://localhost:5000

### 3. 자동화 시스템
- **자동 동기화**: 5분 간격
- **자동 백업**: 매일 자정
- **시스템 모니터링**: 15분 간격
- **부팅 시 자동 시작**: systemd 서비스

## 📁 파일 구조

```
~/gbrain-docker/
├── brain/                    # 옵시디언 원본 파일
├── web/                      # 웹 인터페이스
│   ├── index.html           # 메인 페이지
│   └── app.py               # Flask 백엔드
├── postgres_data/           # PostgreSQL 데이터
├── backups/                 # 자동 백업
├── startup_system.sh        # 시스템 시작 스크립트
├── search_via_docker_fixed.sh # 검색 스크립트
├── import_to_postgres.sh    # 데이터 동기화 스크립트
├── install_service.sh       # 서비스 설치 스크립트
├── monitor_system.sh        # 모니터링 스크립트
├── system_config.json       # 시스템 구성
└── *.log                    # 로그 파일
```

## 🚀 시스템 시작 방법

### 방법 1: 수동 시작
```bash
cd ~/gbrain-docker
./startup_system.sh start
```

### 방법 2: 시스템 서비스 사용
```bash
# 서비스 시작
sudo systemctl start obsidian-search

# 서비스 상태 확인
sudo systemctl status obsidian-search

# 서비스 중지
sudo systemctl stop obsidian-search
```

### 방법 3: 컴퓨터 부팅 시 자동 시작
```bash
# 서비스 활성화 (부팅 시 자동 시작)
sudo systemctl enable obsidian-search
```

## 🔍 사용 방법

### 웹 인터페이스
1. 브라우저에서 http://localhost:5000 접속
2. 검색어 입력 (예: "프로젝트", "VF", "보노하우스")
3. 검색 결과 확인 및 문서 탐색

### 명령어 인터페이스
```bash
# 기본 검색
cd ~/gbrain-docker
./search_via_docker_fixed.sh

# 직접 SQL 검색
docker exec gbrain-postgres psql -U gbrain_user -d gbrain -c "SELECT file_name FROM obsidian_documents WHERE content ILIKE '%프로젝트%';"

# 통계 확인
docker exec gbrain-postgres psql -U gbrain_user -d gbrain -c "SELECT COUNT(*) as 문서수, SUM(word_count) as 총단어 FROM obsidian_documents;"
```

## ⚙️ 시스템 관리

### 데이터 동기화
```bash
# 수동 동기화
cd ~/gbrain-docker
./import_to_postgres.sh

# 또는
./startup_system.sh sync
```

### 백업 관리
```bash
# 수동 백업 생성
./startup_system.sh backup

# 백업 목록 확인
ls -la ~/gbrain-docker/backups/
```

### 로그 확인
```bash
# 시스템 로그
tail -f ~/gbrain-docker/system.log

# 서비스 로그 (systemd)
sudo journalctl -u obsidian-search -f

# 모니터링 로그
tail -f ~/gbrain-docker/monitor.log
```

### 시스템 상태 확인
```bash
# 전체 상태
./startup_system.sh status

# 개별 컴포넌트
docker ps | grep postgres
curl http://localhost:5000/api/system/status
```

## 🔄 자동화 기능

### 1. 자동 동기화
- **간격**: 5분
- **기능**: 옵시디언 파일 변경사항 자동 감지 및 PostgreSQL 반영
- **설정**: `web/app.py`의 `SYNC_INTERVAL` 변수

### 2. 자동 백업
- **시간**: 매일 자정 (00:00)
- **위치**: `~/gbrain-docker/backups/YYYYMMDD_HHMMSS/`
- **내용**: PostgreSQL 덤프 + 구성 파일

### 3. 시스템 모니터링
- **간격**: 15분
- **기능**: PostgreSQL, 웹 API, 데이터 상태 확인
- **문제 감지 시**: 자동 복구 시도 및 로그 기록

## 🛠️ 문제 해결

### 문제 1: PostgreSQL 연결 실패
```bash
# 컨테이너 상태 확인
docker ps | grep postgres

# 컨테이너 재시작
docker restart gbrain-postgres

# 로그 확인
docker logs gbrain-postgres
```

### 문제 2: 웹 인터페이스 접속 불가
```bash
# 웹 서버 프로세스 확인
ps aux | grep app.py

# 웹 서버 재시작
cd ~/gbrain-docker/web
pkill -f app.py
nohup python app.py &
```

### 문제 3: 데이터 동기화 실패
```bash
# 동기화 스크립트 실행
cd ~/gbrain-docker
bash import_to_postgres.sh

# 오류 로그 확인
tail -f ~/gbrain-docker/system.log
```

### 문제 4: 시스템 서비스 오류
```bash
# 서비스 상태 확인
sudo systemctl status obsidian-search

# 서비스 재시작
sudo systemctl restart obsidian-search

# 서비스 로그 확인
sudo journalctl -u obsidian-search -n 50
```
