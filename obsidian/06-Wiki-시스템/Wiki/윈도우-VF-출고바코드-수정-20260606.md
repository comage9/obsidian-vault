# VF 출고 바코드 생성 HTML 수정 내역

**파일:** `E:\coding\VF 출고 바코드\vf 출고 바코드 생성 (1).html`

## 수정 항목

### 🔴 버그 수정 (7건)

1. **CSS 중복 제거**
   - `.container` 내 `flex-direction: column` 중복 선언 제거
   - `@media print` 내 `.barcode-container` position:absolute 규칙 중복 제거

2. **번역 객체 중복 키 제거**
   - `searchFocus`, `processData` 키가 ko/vi 번역 객체에 중복 선언 → 하나만 남김

3. **processBtn 중복 코드 블록 제거**
   - 데이터 처리 후 동일한 `barcodeCountEl` 업데이트/`totalPages` 계산/`currentPage` 초기화/`updatePagination()` 코드가 두 번 연속 실행됨 → 중복 블록 제거

4. **테이블 행 삭제 시 historyData 동기화**
   - `deleteBtn` 클릭 시 `barcodeData`에서만 제거되고 `historyData`에서는 제거되지 않음 → 통계 패널에 삭제된 항목이 계속 누적
   - 수정: `historyData.findIndex()`로 동일 항목 찾아서 `splice()` + `saveHistory()` 호출

5. **빈 셀 채우기 계산 수정**
   - `remainingCells = cellsPerRow - ((endIndex - startIndex) * 2 % cellsPerRow)`
   - 문제: totalCells가 이미 cellsPerRow의 배수이면 `cellsPerRow`개만큼 빈 셀이 추가됨
   - 수정: `(cellsPerRow - (totalCells % cellsPerRow)) % cellsPerRow` — 항상 0~cellsPerRow-1 반환

6. **검색 하이라이트 인덱스 개선**
   - 기존: DOM 순회 + textContent 비교 (구조 변경 시 취약)
   - 수정: `foundIndex % itemsPerPage`로 직접 해당 행 인덱스 계산

7. **바코드 크기 설정 휘발성**
   - `barcodeWidth`, `barcodeHeight`를 localStorage에 저장/불러오도록 개선
   - `loadBarcodeSizeSettings()` 함수 추가
   - `applySizeBtn` 클릭 시 `localStorage.setItem()` 저장

### 🟢 기본 품목 DB 내장 + CSV 업로드 지원

8. **VF 기본 품목 DB 내장 (892개)**
   - `vf_default_products.js` 파일 생성: 쿠팡 발주서 CSV의 바코드→제품명 892개 매핑
   - HTML `<head>`에서 `<script src="vf_default_products.js">` 로 참조
   - `loadProductDB()` 수정: DEFAULT_PRODUCT_DB → localStorage 사용자 데이터 순으로 병합
   - 사용자 업로드 데이터가 기본 DB보다 우선순위 높음

9. **CSV 업로드 지원 추가**
   - `accept=".xls,.xlsx,.csv"` 로 확장
   - CSV 파일 감지 시 텍스트 기반 파싱 (따옴표/콤마 처리)
   - 기존 엑셀 업로드는 그대로 유지
