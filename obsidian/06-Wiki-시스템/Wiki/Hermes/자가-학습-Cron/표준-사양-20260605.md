# Hermes 자가 학습 Cron — 표준 사양 (모든 에이전트 공통)

> **적용 대상**: Windows Hermes / Linux Hermes / 신규 에이전트 (Coupang 자동화, KPP, LS, Supplier Hub, Syncthing, Hermes 자체 등)
> **작성일**: 2026-06-05
> **원안**: 주현 김 (보노하우스)
> **목적**: 1인 운영 + 3개 시스템(LS/KPP/Supplier Hub) 환경에서 AI 네이티브 자가 학습 인프라 표준화

## 1. 핵심 정책 (반드시 준수)

### 1-1. 시간대 정책
- **"자기 전" 시간대만 ❌** → **사용자 명령 외 전 시간대** (낮·밤·새벽 구분 없이)
- **트리거 기준**: 에이전트가 사용자 명령을 **대기 중**일 때 (=idle)면 자가 학습 가능
- **자체 크론 cron schedule**: 분야별 1개씩 시간 안 겹치게 분산 (예: 23:30 / 00:30 / 01:30 / 02:30 / 03:30)

### 1-2. 사용자 명령 우선 정책 (최우선)
- 자가 학습 cron 실행 중 **사용자 명령 도착 시 즉시 중단 + 사용자 지시 우선**
- 진행 중 작업은 **강제 종료 X** → 영구 자원(메모리/DB) commit 후 안전 중단
- 중단 보고 형식: `"사용자 명령 수신, [cron명] 중단 (YYYY-MM-DD HH:MM)"` 1줄
- **이 정책은 모든 다른 정책보다 우선**

### 1-3. 분야당 cron 1개 (분산 배치)
- **단순 등록 같은 작업에 cron 10개 X**
- 분야당 **정확히 1개** cron
- 분야가 늘어나면 cron도 1개씩 추가 (예: 5개 분야 → 5개 cron)
- 분야 분류:
  1. **KPP (WPPS)** — 팔레트 납품/출하 통보 등록
  2. **LS Coupang** — 쿠팡 차량 등록/조회
  3. **Supplier Hub** — 쿠팡 발주서 다운로드
  4. **Syncthing** — 크로스 디바이스 동기화
  5. **Hermes 자체** — agent/mandatory-verification/cron/kanban 등

### 1-4. 외부 시스템 접근 금지
- cron = **문서/스킬/위키 점검만**
- `browser_navigate`, `curl`로 실서비스(ls.coupang.com, wpps.logisall.net, supplier.coupang.com) 직접 조회 금지
- **변경 발견 시점**: 사용자가 실서비스 사용 중 발견 → cron은 **문서/스킬 갱신만**

## 2. cron 표준 정의 (각 분야 공통)

### 2-1. 구조
```json
{
  "name": "<분야>-nightly",
  "schedule": "M H * * *",        // 분산 시간
  "skills": ["mandatory-verification", "<분야 스킬>"],
  "model": "<현재 main model>",
  "deliver": "origin",            // 사용자 home channel
  "repeat": "forever"
}
```

### 2-2. Prompt 템플릿 (모든 분야 공통)

```
오늘 날짜 기준 [<분야>] 자동화 함정/패턴 자가 점검 1회 실행.

[필수 사전 단계]
1. skill_view(name='<분야-스킬>') 로드 — SKILL.md §Pitfalls + references/*.md 읽기
2. Wiki E:\hermes-backup\obsidian\06-Wiki-시스템\Wiki\<분야폴더>\ 의 모든 .md 읽기
3. **외부 시스템 직접 조회 금지** (cron은 문서/스킬 점검만)

[판단 단계]
4. 함정/패턴/규칙 변화 발견 시:
   - <분야>/SKILL.md §Pitfalls에 append
   - Wiki E:\hermes-backup\obsidian\06-Wiki-시스템\Wiki\<분야폴더>\ 에 새 파일 (검증 출처 명시)
   - 메모리 갱신: 사용자 선호/절차 사실만 (작업 진행 상황 X)
5. 변경 없으면 "<분야> 점검 완료: 변경 없음 (YYYY-MM-DD)" 1줄
6. **사용자 명령 도착 시 즉시 중단** → "사용자 명령 수신, <분야> cron 중단 (YYYY-MM-DD HH:MM)" 1줄

[진행보고]
- (a) 점검 대상 (b) 비교 방법 (c) 검증 결과 (변경 파일 경로/크기 또는 "변경 없음")

상세: E:\hermes-backup\obsidian\06-Wiki-시스템\Wiki\Hermes\자가-학습-Cron\<분야>-Nightly-YYYYMMDD.md 참조
```

### 2-3. 검증 체크리스트 (cron 첫 실행 후)
- [ ] cron이 skill + wiki만 읽고 외부 시스템 미접근
- [ ] 변경 발견 시 양쪽(스킬+위키) 동시 갱신
- [ ] 사용자 명령 시 즉시 중단 보고
- [ ] 0건 변경 시 한 줄 보고 (verbose X)
- [ ] 스킬/위키 파일 크기 증가분 검증

## 4. 운영원칙 (모든 에이전트 공통)

### 4-1. 4-Step 의무 (변경 없음, 강화)
1. 위키 검색 먼저
2. 스킬 로드 필수
3. 메모리/프로필 매 턴 참조
4. 완료 후 위키 저장 (E:\hermes-backup\obsidian\06-Wiki-시스템\Wiki\)

### 4-2. 절대 금지 A~F (변경 없음)
- (A) 추측 금지
- (B) 거짓말 금지
- (C) 실제 검증 의무
- (D) "됐다" 단독 금지
- (E) 진행보고 a/b/c (무엇/어떻게/검증)
- (F) 수행 후 저장 의무

### 4-3. 동기화 정책 (Syncthing)
- E:\hermes-backup\ = Windows ↔ Linux 공유 저장소 마스터
- Linux 마운트: `/media/comage/data1/hermes-backup/` (NOT data/)
- 초기 sendonly → 데이터 검증 후 sendreceive 전환
- `.stfolder` 수동 생성 금지
- 모든 자가 학습 cron 결과/사양 = 이 표준 사양 1개로 통일 (분야별 추가 사양 가능)

### 4-4. 진행보고 형식
- **단답+핵심표** 선호
- **3가지**: (a) 무엇 (b) 어떻게 (c) 검증
- "?" = stop
- 모르면 URL 직접 물을 것
- 음성인식 오류 인지 (사모차=3호차, 추라=출하, 사무차=4호차)

## 5. 신규 에이전트 온보딩 시 체크리스트

새 에이전트가 합류하면 다음 순서로:

1. **이 표준 사양 파일** 1개만 받기 → `자가-학습-Cron-표준-사양-20260605.md`
2. 분야별 cron 1개씩 생성 (시간 분산)
3. 분야별 점검 대상 스킬/위키 1개 이상 사전 확인 (없으면 신규)
4. 사용자 명령 우선 정책 테스트 (진행 중 명령 입력 → 즉시 중단)
5. 첫 실행 결과 보고 (변경 없음 1줄 OR 변경 파일 경로)

**이온보딩을 다른 에이전트(Linux, 신규)에게 그대로 전달.**
