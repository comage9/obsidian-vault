# Telegram 운영 원칙 동기화 설정

## 개요
CLI 세션(현재 Windows Hermes)의 운영 원칙(4-Step 의무, 절대금지 A~F)을 Telegram 세션에도 동일하게 적용.
Linux Hermes(우분투)의 Telegram도 동일한 방식 적용 가능.

## 적용 방식 (2단계)

### 1단계: telegram-core-rules 스킬 생성 ✅
- 스킬명: `telegram-core-rules` (Windows: `~/.hermes/skills/dogfood/telegram-core-rules/SKILL.md`)
- 내용: CLI의 mandatory-verification + MEMORY.md 운영 원칙을 Telegram에 맞게 포팅
- 설치 확인: `hermes skills list | grep telegram-core-rules`

### 2단계: channel_prompts 설정 ✅
- `config.yaml`의 `telegram.channel_prompts` 에 홈채널(5708696961) 프롬프트 추가
- 내용: "모든 작업 시작 전 /skill telegram-core-rules 를 로드하고 4-Step 의무를 따를 것"
- Telegram 세션이 새로 시작될 때마다 이 프롬프트가 시스템 메시지로 주입됨

## 동작 방식

```
사용자 메시지 → Telegram 게이트웨이
  → config.yaml channel_prompts[5708696961] 주입
    → 시스템 프롬프트에 "[필수] 모든 작업 시작 전 /skill telegram-core-rules..."
      → Telegram 에이전트가 /skill telegram-core-rules 실행
        → 4-Step 의무 + 절대금지 A~F 적용
```

## 적용 범위

| 항목 | CLI | Telegram |
|:-----|:---:|:--------:|
| 4-Step 의무 (위키→스킬→메모리→저장) | ✅ | ✅ (channel_prompts+skill) |
| 절대금지 A~F | ✅ | ✅ (skill 내장) |
| 위키 검색 먼저 | ✅ | ✅ |
| 질문=답만, 동시실행 금지 | ✅ | ✅ |
| 단답+핵심표 | ✅ | ✅ |
| 완료 후 위키 저장 | ✅ | ✅ |
| MEMORY.md / USER.md | ✅ (매턴 주입) | ✅ (공유) |

## 한계
- 스킬 로드는 `/skill telegram-core-rules` 명령어에 의존 — 에이전트가 이를 수행해야 함
- channel_prompts는 홈채널(5708696961)만 대상 — 다른 채널/그룹별 별도 설정 필요
- Telegram Gateway 재시작(`hermes gateway restart`) 후 설정 적용

## Linux Hermes 적용 방법
```bash
# 1. telegram-core-rules 스킬 설치
hermes skills install https://raw.githubusercontent.com/.../telegram-core-rules/SKILL.md

# 2. channel_prompts 설정
hermes config set telegram.channel_prompts '{"5708696961": "[필수] 모든 작업 시작 전 /skill telegram-core-rules ..."}'

# 3. Gateway 재시작
hermes gateway restart
```
