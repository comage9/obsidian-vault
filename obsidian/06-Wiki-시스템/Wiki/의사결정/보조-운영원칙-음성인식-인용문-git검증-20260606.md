# 보조 운영 원칙

> MEMORY.md 용량 한계로 분리된 보조 항목 3건

## 1. 음성인식 오류 인지

주현 님이 음성 입력 시 특정 단어가 자주 오인식됨.

| 발화 | 실제 의도 |
|------|---------|
| **사모차** | **3호차** (LS 트럭오더 호차 번호) |
| 기타 음성 오인식 가능 → 맥락으로 의도 추론 |

**대응:** 음성인식 오류로 보이는 입력은 맥락 기반 의도 추론 후 "OOO 말씀이시죠?" 확인

## 2. 위반 패턴 인용문 (사용자 실제 발언)

| 발언 | 의미 | 대응 |
|------|------|------|
| **"하지도 않고 거짓말하지마"** | 실행 안 하고 "했다"고 보고 금지 | 모든 완료 보고는 실제 도구 호출 출력으로 증명 |
| **"안되는거된다고하지마"** | 불가능한 것을 가능하다고 말하지 마 | 모르겠으면 "모르겠습니다" / "자동화 안 됩니다" |
| **"안다고생각하지말고항상확인검증"** | 추측 금지, 항상 실제 확인 | curl/stat/read_file로 검증 후 보고 |
| **"이전 생성 방법이 있는데?"** | 기존 성공 방식을 무시하고 새 방식 시도 금지 | 기존 절차/스킬 우선 적용 |

## 3. 운영원칙 5단계 (Git State 검증)

기존 4단계에 Git State 검증을 5단계로 추가:

| 단계 | 내용 | 설명 |
|:----:|------|------|
| 1 | 위키 검색 | `search_files`로 관련 문서 검색 |
| 2 | 스킬 로드 | `skill_view()`로 정독 후 적용 |
| 3 | 메모리/프로필 참조 | 매 턴 MEMORY/USER 확인 |
| 4 | Git State 검증 ⭐신규 | Git에 최신 상태인지 `git status`/`git log` 확인. **없다고 단정 전 반드시 확인** |
| 5 | 완료 후 위키 저장 | `E:\hermes-backup\obsidian\06-Wiki-시스템\Wiki\` 하위에 저장 |

**4단계 상세 규칙:**
- "파일/디렉토리가 없다" → `git ls-files`로 Git에 있는지 먼저 확인
- "스킬이 없다" → `skills_list()` + 로컬 skills/ 디렉토리 ls 확인
- Git에 있는데 로컬에 없으면 → `git pull`
- **절대 금지:** `git pull` 안 하고 "없다"고 보고
