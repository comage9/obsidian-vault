# Windows-Linux 공유 백업 협약

## 배경
Windows Hermes와 Linux Hermes(우분투) 간 크로스 디바이스 협업을 위해 공유 저장소를 정의.

## 저장소 경로
| 기기 | 경로 | 비고 |
|:----|:-----|:-----|
| **Windows** | `E:\hermes-backup\` | 물리 디스크 E: |
| **Linux** | `/media/comage/data1/hermes-backup/` | 외부 디스크 마운트 (⚠️ `/media/comage/data/` 아님! data1) |
| 동기화 | Syncthing 양방향 | 초기 sendonly → 검증 후 sendreceive |

## 디렉토리 구조
```
hermes-backup/
├── hermes-snapshot-YYYYMMDD-HHMMSS/   # 전체 스냅샷
│   ├── config/
│   ├── memories/
│   ├── obsidian/
│   ├── scripts/
│   └── skills/
├── memories/                           # MEMORY.md + USER.md
├── obsidian/                           # Obsidian 볼트
│   └── 06-Wiki-시스템/Wiki/            # 공유 위키 (작업 사항/Pitfalls 저장)
│       ├── 의사결정/                    # 설정/협약/가이드
│       └── 물류/                       # KPP/LS/서플라이허브
└── skills/                             # 스킬 백업
```

## 접두사 규칙 (파일명)
| 접두사 | 용도 |
|:------|:------|
| `윈도우-` | Windows 전용 설정/가이드 |
| `리눅스-` | Linux 전용 설정/가이드 |
| (접두사無) | 양쪽 공용 문서 |

## 새 작업 사항 기록 의무
새로운 절차/Pitfalls/에러 해결책/설정 변경 발생 시:
1. 즉시 `E:\hermes-backup\obsidian\06-Wiki-시스템\Wiki\` 하위 적절한 폴더에 .md 저장
2. 접두사 규칙 준수 (공용/윈도우-/리눅스-)
3. Linux는 `/media/comage/data1/hermes-backup/` 경로 사용

## Syncthing 주의사항
- **초기 설정: 무조건 sendonly** — 데이터 검증 후 sendreceive 전환
- `.stfolder` 수동 생성 **절대 금지** — 데이터 전소 위험
- 데이터 손실 시 Git 자동 Push(6시간)가 복구 생명선
- 치명적 경험: sendreceive에서 "folder marker missing" 오류 시 빈 폴더 기준 동기화로 1GB+ 데이터 소실

## Windows Hermes 특이사항
- gateway: `SystemError: AST constructor recursion depth mismatch` 발생 시 환경변수 `PYTHONRECURSIONLIMIT=3000` 설정
- Chrome remote debugging: `--user-data-dir=<temp>` 플래그 **필수**
