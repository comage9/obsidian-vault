## 📋 0. 이 문서가 해결하는 문제

**매번 처음 대하듯 행동하는 문제** — 곰너이 님 시스템의 위키/스킬/MEMORY는 잘 구축되어 있으나, **자동 recall(메모리 자발적 검색)이 빠져 있어서** 같은 작업을 반복할 때도 처음부터 다시 시작.

**표준 long-term memory 시스템 (mem0 57,988★, Letta 23,202★, MemGPT)**은 자동 recall이 기본 제공. 곰너이 님 시스템도 동일하게 만들어야 함.

## 🎯 1. 적용 규칙 — 모든 에이전트 공통 (2026-06-08 곰너이 확정)

### §0 Tool-First Auto-Recall — 5단계 의무보다 먼저 실행

**사용자 지시 수신 즉시, 어떤 답변/실행/판단보다도 먼저** 아래 3개 도구를 **무조건, 병렬로** 호출.

```yaml
trigger: on_user_message
mode: parallel
tools:
  - search_files(pattern=<키워드>, path=<Wiki 루트>, target='files')
  - skills_list()
  - session_search(query=<키워드>, limit=3, sort='newest')
```

### §0 결과 처리

| 결과 | 다음 동작 |
|------|----------|
| Wiki 매치 | `read_file` → 답변에 인용 |
| Skill 매치 | `skill_view(name)` → 절차 따르기 |
| Session 매치 | `session_id` + `around_message_id` → 컨텍스트 복원 |
| **모두 0건** | **"이전에 다뤄본 적 없음. 처음 진행하겠습니다" 명시** |
| **0건인데 "확인했습니다" 보고** | **거짓말. 절대 금지** |

### §1 5-Step 의무 (모든 작업 시작 전)

```
Step 1: 위키 검색 — search_files (안 하면 위반)
Step 2: 스킬 로드 — skill_view() 정독
Step 3: 메모리/프로필 매 턴 참조
Step 4: Git State 검증 — git status/git ls-files
Step 5: 완료 후 위키 저장
```

### §2 절대 금지 A~F + 추가 금지 9-10

```
A 추측 금지 — 모르면 "모르겠습니다"
B 거짓말 금지 — "했다"고 말한 건 실제 도구 호출로 증명
C 실제 검증 의무 — curl/stat/read_file (안 하면 = 거짓말)
D "됐다" 단독 금지 — 변경 내역 증빙 필수
E 진행보고 a/무엇을 b/어떻게 c/검증
F 수행 후 저장 의무 — 위키 기록 안 함 = 미완료

🚨 9 (2026-06-08): Tool-First Auto-Recall 건너뛰기 = 거짓말
🚨 10 (2026-06-08): "확인했다/검증했다" 답변 시 도구 호출 이력 없이 보고 = 거짓말
```

### §3 3중 완료 의무 (2026-06-08 곰너이 확정)

모든 작업 완료 시 (a) Wiki 기록 (b) README 갱신 (c) Git push 3종 모두 수행. 부분 완료 보고 금지.

### §4 카르파티 LLM Wiki 3-Layer / 3-Operation (2026-06-08)

**3-Layer:**
- Raw Sources (불변) — LLM이 읽기만, 수정 안 함
- The Wiki (LLM 관리) — obsidian vault
- The Schema — SOUL.md + USER.md + MEMORY.md

**3-Operation:**
- Ingest — 한 작업으로 10-15 페이지 동시 갱신 가능
- Query — 자동 recall (§0) → 합성 답변
- Lint — 자가학습 cron + 정기 점검

## ⚠️ 6. 곰너이 님 핵심 인용 (절대 잊지 말 것)

> "이전에도 답변은 뭘 하든 항상 확인한다고 했는데, **확인이 안 된다는 게 문제**지."

> "어떤 지시사항에 대해서 자꾸 **혼자 판단**하고 기존에 했던 일인데도 **매번 처음 하는 것처럼 처리**하는 문제가 있다."

> "이미 경험했던 상황이라면 **스킬화되어 있는 것을 그대로 사용**하거나 기존 스킬을 로드하면 되는데, 그걸 하지 않고 매번 처음 대하듯 행동하니까 문제가 되는 거지."

> "**위키 LLM 시스템이 구축되어 있어서 이걸 검색만 해도 했던 일을 반복하는 비효율을 줄일 수 있단 말이야.** 그게 잘 안돼서 여러 가지 운영 원칙과 Soul MD 같은 다른 사항들도 줄줄이 만들어 놨는데, **전혀 활용이 안 되고 있다는 건가?**"

→ **§0 Tool-First Auto-Recall이 직접 해결. 단순 약속이 아니라 도구 호출 강제.**
