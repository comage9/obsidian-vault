# Ingest Template — Source → Wiki

> **용도:** Sources 폴더에 새 원본 자료가 들어왔을 때, AI가 읽고 Wiki 문서로 정리

## 트리거 문구
```
/ingest
/ingest [파일명 또는 주제]
Sources 읽고 Wiki 정리해줘
```

## 실행 절차

### Step 1: Sources 확인
```bash
ls Wiki/Sources/텍스트/  # 새로 들어온 파일 확인
```

### Step 2: AI에게 전달할 프롬프트

```markdown
Sources 폴더에 새 자료가 있습니다. 다음 작업을 수행해 주세요:

1. Sources/텍스트/ 경로의 모든 파일을 읽으세요
2. 내용을 분석하여 Wiki/ 아래 적절한 폴더에 주제별 문서를 만드세요
   - 의사결정/ → 설정/협약/가이드 성격
   - 물류/ → KPP/LS/VF/서플라이허브 관련
   - 운영원칙/ → 에이전트 행동 규칙
3. 각 문서에 다음을 포함하세요:
   - 생성일: [날짜]
   - 출처: [참고한 원본 파일명]
   - 태그: [키워드]
4. 중복되는 내용은 하나의 문서로 합치세요
5. 기존 관련 문서가 있으면 [[wikilink]]로 연결하세요
```

### Step 3: 결과 저장
- 생성된 문서 → `Wiki/의사결정/` 또는 `Wiki/물류/` 등
- ingest 내역 → `log.md`에 추가
- index.md 갱신 (새 페이지 반영)

## 예시

**사용자:** `/ingest ls-api-20260608.json`

**내 수행:**
1. `Sources/텍스트/LS/ls-api-20260608.json` 읽기
2. 내용 분석 → LS 차량 등록 현황 파악
3. `Wiki/물류/쿠팡/LS/2026-06-08-차량등록-현황.md` 생성
4. 출처 명시, 기존 문서와 링크 연결
5. `log.md`에 기록, `index.md` 갱신
