# Git 단일 브랜치 정책 (master only)

**날짜:** 2026-06-06
**분류:** 운영원칙 > Git 정책
**대상:** Windows Hermes (CLI) + Linux Hermes

## 정책 (확정)

**`obsidian-vault` 원격 저장소는 `master` 브랜치 단일로 운영한다.**

- ✅ 기본 브랜치: `master` (origin HEAD)
- ✅ 모든 push: `master`로
- ❌ `main` 브랜치: 사용 금지
- ❌ 새 브랜치 생성 금지
- ❌ `main` 브랜치 재발생 시: 즉시 삭제

## 적용 범위

| 원격 저장소 | URL | 브랜치 정책 |
|-------------|-----|-------------|
| `obsidian-vault` | https://github.com/comage9/obsidian-vault.git | `master` 단일 |

## 검증 명령

```bash
# 원격 브랜치 목록
git ls-remote --heads origin
# 기대 출력: refs/heads/master 단일

# 로컬 브랜치
git branch -a
# 기대 출력: master + remotes/origin/master

# 기본 브랜치 확인
git symbolic-ref refs/remotes/origin/HEAD
# 기대 출력: refs/remotes/origin/master
```

## main 브랜치 정리 이력 (2026-06-06)

**배경:**
- `main` 브랜치는 "Wiki auto-sync" 자동화 흐름으로 별도 운영 (2026-05-31 마지막 커밋)
- 사용자 Windows Hermes 작업은 모두 `master`에 커밋
- 두 브랜치는 내용 중복 없음 (별도 흐름)

**조치:**
- 원격 `main` 삭제: `git push origin --delete main` → `[deleted] main`
- 로컬 추적 제거: `git remote prune origin`
- 결과: 원격 = master 단일, 로컬 = master + origin/master

**판단 근거:**
- 사용자 작업이 모두 `master`에 있음
- `main`의 "Wiki auto-sync" 커밋들은 master와 중복되는 별도 자동화 bot 작업
- 삭제해도 master에 영향 없음 (검증 완료)

## push 규칙 (Pre-flight Check 연동)

1. 작업 완료 후 `sync_hermes_to_backup.py` 실행
2. 스크립트가 `master`로 자동 push
3. **브랜치 지정 없는 push 금지** (스크립트 외 수동 push 시 `git push origin master` 명시)

## 위반 시 대응

- `main` 브랜치 발견 시 → 즉시 `git push origin --delete main`
- 로컬에 master 외 브랜치 생성 시 → `git branch -D <branch>` 후 master로 통합
