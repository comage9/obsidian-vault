# LS → KPP 일일 운영 흐름 (2026-06-03 사용자 진술)

**목적**: LS에서 등록한 차량 정보를 PDF/Tracking으로 추출 → KPP에 출하 통보 등록

## 5단계 운영 흐름

### 1단계: LS 오늘자 등록 확인

- **시점**: 매일 오전 (오후 1시 차량요청 전)
- **API**: `GET https://ls.coupang.com/api/v2/truckOrder?orderDate=YYYYMMDD` 또는 `GET /truckOrderTracking`
- **확인 항목**: CONFIRMED/SUBMITTED 상태 차량 목록
- **저장 위치**: PostgreSQL `ls_vehicle_snapshot` 테이블

### 2단계: 3대 등록 (등록 안 됐을 때만)

- **시점**: 오후 1시쯤 (쿠팡 차량 요청 시간)
- **조건**: 오늘 자 등록 차량 0건일 때
- **3대 고정**: 90626 / 90628 / 90269
- **API**: `POST https://ls.coupang.com/api/v2/truckOrder/templates/batch/creation/{orderDate}` (배치)
- **응답**: `data: [{truckRequestId: 27375860, ...}, ...]`

### 3단계: 3~4시 재확인 + PDF 다운로드

- **시점**: 오후 3~4시
- **목적**: 2단계 후 ALC_STATUS 변경 확인 + 차량 정보 확정
- **PDF endpoint**: `GET https://ls.coupang.com/linehaul/slip/generate?truckRequestId={id}&locale=ko_KR`
- **Accept 헤더**: `application/json,application/pdf`
- **저장 위치**: `/home/comage/다운로드/LinehaulSlip-{truckRequestId}.pdf`

### 4단계: 차량 정보 추출

- **PDF 파싱**: `pdftotext -layout LinehaulSlip-27375860.pdf` → 운행일자/차량번호/트럭바코드/성함/연락처
- **톤수 확인**: `GET /truckOrderTracking` 응답의 `truckType.name` (예: "5T", "11T")
- **DB 저장**:
  - `vehicles` 테이블: 차량번호(숫자만)/트럭바코드
  - `drivers` 테이블: 성함/연락처
  - `vehicle_driver_assignments` 테이블: vehicle_id ↔ driver_id
- **케이스 분류** (자동):
  - **케이스 1**: 같은 차량+전화, 다른 이름 → 교대
  - **케이스 2**: 같은 차량, 다른 이름+전화 → 임시/대체

### 5단계: KPP PBM140MW 출하 통보 등록

- **시점**: 4단계 직후
- **API**: `POST https://wpps.logisall.com/ps/PBM140MW.save`
- **팔레트 규칙**:
  - **5T** = 12팔레트
  - **11T** = 14팔레트
  - 14T = 17~18팔레트 (확인 필요 시 사용자 확인)
- **차량번호 규칙**: 숫자만 6자리 (한글 제외) — [출하통보-PBM140MW-차량번호-규칙](./../KPP/출하통보-PBM140MW-차량번호-규칙.md) 참조
- **응답**: `flag:true` = 성공

## 자동화 스크립트

- **KPP 자동 등록 스크립트**: 미작성 (사용자 수동 트리거 선호, "한 번에 한 가지만")

## Pitfalls

1. **LS `/order` API는 차량번호/톤수 미포함** — CONFIRMED 상태에서도 `truckInfo` 비어있음 → PDF + Tracking API 보강 필수
2. **3대 등록 안 됐는데 batch creation 누락** → 3~4시 PDF 단계에서 빈 결과
3. **PDF endpoint 404**: `Accept` 헤더 누락 시
4. **PBM140MW 차량번호 6자리 규칙** 무시 시 "한글은 2자로 인식" 에러
5. **PDF 파싱 한글 깨짐**: `pdftotext` 인코딩 옵션 필요 시 `-enc UTF-8`
