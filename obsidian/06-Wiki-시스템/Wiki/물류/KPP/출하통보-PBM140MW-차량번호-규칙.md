# PBM140MW 출하통보 — 차량번호 입력 규칙

**최종 확정일**: 2026-06-03
**에러 사례**: `flag:false, message:"차량번호는 6자리만 가능합니다. (한글은 2자로 인식합니다!)"`

## 규칙

KPP PBM140MW는 **차량번호 6자리만** 허용:
- 한글이나 문자 포함 시 한 글자를 2자로 카운트
- 예: `경기95자6464` = 한글 2자 + 숫자/문자 7자리 = 9자리로 카운트 → 거부
- **정답**: 숫자만 추출 → 6자리로 만든다

## 변환 절차

| 원본 (LS/PDF 추출) | 6자리 변환 | 비고 |
|---|---|---|
| `경기95자6464` | `956464` | 1호차 90626 |
| `경기89바1454` | `891454` | 2호차 90628 |
| `충북90아6169` | `906169` | 3호차 90269 |

**변환 로직 (정규식)**: `re.sub(r'[^0-9]', '', 차량번호)` → 결과는 가변 길이 → 6자리로 자르거나 종량 처리

## 함정 (Pitfalls)

1. **원본 차량번호 그대로 입력 시 거부**: 한글 2자 카운트 → "한글은 2자로 인식합니다" 메시지
2. **숫자만 추출 후 길이 검증 필수**: 가끔 PDF 파싱 오류로 5자리/7자리 나옴
3. **에러 응답 메시지 정확히 인식**: "6자리만 가능" → 길이 문제, "형식" → 다른 문제

## 적용 워크플로우

1. LS에서 PDF/Tracking 정보 추출 → 차량번호 원본 확보
2. Python `re.sub(r'[^0-9]', '', car_num)` 으로 숫자만 추출
3. 6자리 검증 (`len(car_num_digits) == 6`)
4. PBM140MW 페이로드의 `car_num` 필드에 입력
5. POST 후 `flag:true` 확인
