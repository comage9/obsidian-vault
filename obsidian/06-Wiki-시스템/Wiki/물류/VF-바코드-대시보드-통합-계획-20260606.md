# VF 바코드 시스템 ↔ VF 대시보드 완전 통합 계획

## 1. 현재 상태 분석

### 현재 구성
| 컴포넌트 | 위치 | 형태 |
|---------|------|------|
| VF 대시보드 | `E:\coding\VF-new - 복사본\` | React (Vite + wouter) + Django 백엔드 |
| 바코드 페이지 | `frontend/client/public/barcode/index.html` | **독립 HTML파일** (React 외부, Vite public 폴더) |
| 연결 방식 | 사이드바 `externalUrl` → `http://bonohouse.p-e.kr:5174/barcode/index.html` | 외부 링크 (iframe X) |
| API | `POST /api/delivery/hourly`, `POST /api/baco/transfer-stats` | Django REST (공유) |

### 문제점
- **React 컴포넌트 아님**: 상태 관리(delivery 데이터, 차트)와 대시보드 연결 불가
- **외부 URL 의존**: 독립 실행 시 대시보드 인증/세션 접근 불가
- **중복 데이터 로드**: 바코드 페이지가 별도로 제품 DB(내장 JSON)를 로드 → 대시보드 DB와 이중 관리
- **차트/필터 독립**: 대시보드의 시간대별 차트와 바코드 페이지 필터가 별도 로직

## 3. 세부 통합 계획 (3단계)

### Phase A: React 컴포넌트화 ⏱ 2-3시간

**목표:** 독립 HTML → React 컴포넌트로 전환

| 작업 | 내용 | 상세 |
|:----|:-----|:-----|
| A-1 | `pages/barcode-generation.tsx` 생성 | 기존 `index.html`의 HTML → JSX, CSS → Tailwind/emotion |
| A-2 | 상태 관리 통합 | `useDeliveryData` 훅 → 대시보드 API 재사용 |
| A-3 | 사이드바 교체 | `externalUrl` → `path: "/barcode"` + React Router |
| A-4 | 제품 DB 통합 | 내장 JSON → 대시보드 DB `/api/products` 조회 |

**핵심 전환 항목:**
- `autoExtractCategory()` → 백엔드 API의 category 컬럼 사용
- Barcode js 라이브러리 → React 컴포넌트 래퍼
- localStorage 데이터 → React Context/Query

### Phase B: 기능 통합 ⏱ 2-3시간

**목표:** 대시보드 기능과 양방향 연결

| 작업 | 내용 | 상세 |
|:----|:-----|:-----|
| B-1 | 시간대별 데이터 공유 | 바코드 전송 결과가 `delivery-overview` 차트에 즉시 반영 |
| B-2 | 카테고리 필터 공유 | 바코드 페이지의 분류 필터 → 대시보드 전체 카테고리와 통일 |
| B-3 | 엑셀 다운로드 통일 | 바코드 DB 다운로드 → 대시보드 공용 다운로드 버튼 |
| B-4 | 통계 페이지 연동 | 바코드 출고 통계 → delivery-overview 통계에 병합 |

### Phase C: 고도화 ⏱ 선택

| 작업 | 내용 |
|:----|:-----|
| C-1 | 실시간 바코드 스캔 연동 (웹캠/모바일) |
| C-2 | 인쇄 템플릿 커스터마이징 (대시보드 설정에서) |
| C-3 | 출고 데이터 → 생산 계획 자동 반영 |

## 4. 기술 상세

### 컴포넌트 구조 (Phase A 결과)

```
frontend/client/src/
├── components/
│   ├── barcode/
│   │   ├── BarcodeGenerator.tsx      ← 바코드 생성 + 프리뷰
│   │   ├── BarcodeListView.tsx       ← 생성된 바코드 목록
│   │   ├── CategoryFilter.tsx        ← 분류 필터 (기존 공유)
│   │   ├── DeliveryTimeChart.tsx     ← 시간대별 전송 차트
│   │   └── ProductDBUploader.tsx     ← 제품 엑셀 업로드
├── hooks/
│   ├── useBarcodeGeneration.ts       ← 바코드 생성 로직
│   ├── useProductDB.ts              ← 제품 DB 조회 (대시보드 API)
│   └── useDeliveryStats.ts          ← 시간대별 출고 통계
├── pages/
│   └── barcode-generation.tsx       ← 바코드 페이지 전체
```

### 데이터 흐름

```
[제품 엑셀 업로드] → POST /api/products/upload
                         ↓
                   DB (SQLite)
                         ↓
[BarcodeGenerator] ← GET /api/products?category=X
                         ↓
              [바코드 생성 + 출고 전송]
                         ↓
              POST /api/delivery/hourly
                         ↓
              [차트 실시간 갱신]
```

### API 변경 사항 (필요시)

| 현재 | 변경 제안 |
|------|----------|
| `GET /api/delivery/hourly?days=N` | 동일 (재사용) |
| `GET /api/baco/products` (신규) | 제품 DB + 분류 목록 반환 |
| `GET /api/baco/categories` (신규) | 분류 목록만 반환 (필터용) |

## 5. 리스크

| 리스크 | 대응 |
|-------|------|
| `index.html` 173KB, 2,200줄 → React 변환 작업량 큼 | Phase A에서 가장 핵심 기능(생성/전송/필터)만 우선 변환, 나머지 점진 이관 |
| 대시보드 기존 API와 바코드 페이지 API endpoint 충돌 | `/api/baco/*` 네임스페이스로 격리 (현재 유지) |
| Barcode.js 라이브러리 React 호환성 | `useEffect` + ref로 DOM 기반 라이브러리 래핑 |
| VPN 환경에서 CDN 의존 | 로컬 js 파일 번들링 (Vite 정적 import) |
