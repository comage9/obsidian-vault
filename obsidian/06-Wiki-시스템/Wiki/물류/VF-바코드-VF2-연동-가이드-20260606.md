# VF 바코드 페이지 — VF2 연동 설명서

> **목적:** 다른 에이전트가 VF2(Go 백엔드) 프로젝트에 바코드 페이지를 추가할 수 있도록 현재 적용 방식과 소스 코드를 전달합니다.

---

## 1. 현재 VF 대시보드 적용 방식 (참고)

### 1-1. 파일 배치

```
VF-new - 복사본/
└── frontend/client/public/
    └── barcode/
        └── index.html          ← 독립 HTML (173KB, 2,270줄)
```

**Vite public 폴더**에 배치 → 빌드 시 정적 파일로 자동 복사됨.
`http://localhost:5174/barcode/index.html`로 직접 접근 가능.

### 1-2. 사이드바 연결

**`frontend/client/src/components/sidebar.tsx`**:
- `SidebarItem` 인터페이스에 `externalUrl?: string` 필드 추가
- `externalUrl`이 있으면 `<a>` 태그로 외부 링크 렌더링

**`frontend/client/src/pages/dashboard.tsx`**:
```typescript
{
  key: "barcode",
  path: "/barcode",
  label: "바코드 생성",
  icon: "fa-qrcode",
  description: "송장번호/제품 바코드 생성 및 시간대별 출고 데이터 전송",
  externalUrl: "http://bonohouse.p-e.kr:5174/barcode/index.html",
}
```

### 1-3. 현재 방식의 한계

- **React 컴포넌트 아님** → 인증/세션 공유 불가
- **외부 URL 의존** → 포트/호스트 변경 시 수동 수정
- **별도 API 호출** → 대시보드 API 주소를 하드코딩 (`http://bonohouse.p-e.kr:5176`)

## 2. VF2 (Go 백엔드) 권장 적용 방식

### 2-1. 정적 파일 서빙 (Go)

```go
// main.go 또는 router 설정
r := chi.NewRouter()

// 바코드 페이지 정적 파일 서빙
r.Handle("/barcode/*", http.StripPrefix("/barcode/",
    http.FileServer(http.Dir("frontend/public/barcode"))))

// 또는 임베디드
// go:embed frontend/public/barcode/*
var barcodeStatic embed.FS
r.Handle("/barcode/*", http.FileServer(http.FS(barcodeStatic)))
```

### 2-2. 소스 코드 출처

| 항목 | 위치 | 비고 |
|------|------|------|
| 바코드 페이지 HTML | `github.com/comage9/VF-barcode` | `vf 출고 바코드 생성 (1).html` |
| 최신 버전 | `7d7567e` | 분류 시스템 개선 완료 |
| API 주소 | 내부 하드코딩 | VF2 API 주소로 변경 필요 |

### 2-3. 수정 필요 사항 (VF2 적용 시)

**파일 내 하드코딩된 API 주소 변경:**
```javascript
// AS-IS (현재)
const API_BASE = 'http://bonohouse.p-e.kr:5176';

// TO-BE (VF2)
const API_BASE = '/api';  // 동일 오리진
```

**구현해야 할 API endpoints:**
| Endpoint | Method | 용도 |
|----------|--------|------|
| `/api/delivery/hourly?days=N` | GET | 시간대별 출고 현황 |
| `/api/delivery/hourly` | POST | 시간대 데이터 전송 |
| `/api/baco/transfer-stats` | POST | 출고 기록 전송 |

## 3. 바코드 페이지 핵심 기능 설명

### 3-1. 데이터 구조

**송장 데이터 파일 형식 (CSV 업로드):**
| 열 | 인덱스 | 내용 |
|:--:|:------:|------|
| F | 4 | 바코드 |
| G | 5 | 제품명 |
| H | 6 | **분류 (대분류)** — CSV G열 |

**제품 DB (내장 JSON):**
```javascript
const DEFAULT_PRODUCT_DB = {"R003750270003":"모던플러스 내츄럴오크...", ...};  // 892개
const DEFAULT_PRODUCT_CATEGORY = {"R003750270003":"모던 플러스", ...};         // 893개, 43개 분류
```

### 3-2. 주요 함수

| 함수 | 위치 (줄) | 기능 |
|------|:---------:|------|
| `loadProductDB()` | ~786 | 제품 DB 로드 (localStorage → DEFAULT → autoExtract 순) |
| `autoExtractCategory()` | ~402 | 제품명 → 분류 추출 (fallback 전용) |
| `handleFileUpload()` | ~858 | CSV/엑셀 업로드 파싱 |
| `generateBarcodes()` | — | 바코드 이미지 생성 (JsBarcode) |
| `sendHourlyData()` | — | 시간대별 출고 데이터 전송 |
| `sendTransferStats()` | — | 출고 통계 전송 |
| `downloadExcel()` | ~1020 | 엑셀 다운로드 (SheetJS) |

### 3-3. 분류 시스템 (2026-06-06 개선)

```
우선순위:
1. 사용자 업로드 CSV/엑셀 G열 (대분류) → localStorage
2. DEFAULT_PRODUCT_CATEGORY (CSV 기반 내장, 893개)
3. autoExtractCategory() (제품명 키워드 기반, 최후 fallback)
```
