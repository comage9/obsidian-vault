## 2. LS 주간 운영 상세 (3개 크론)

### 2.1 13:00 — LS 차량 등록 + PDF 출력 + KPP 팔레트 등록

**Phase 1: LS VF67_H 차량 등록 확인**
- LS API 조회: `GET /truckOrderTracking` (VF67_H, 오늘 날짜)
- 이미 등록됨 → 차량 목록 정리만
- 미등록 → 기본 3대 등록
- 통보 형식: `[VF67 - {날짜} 배차 현황]`

**Phase 1.5: PDF 다운로드 + Canon G2010 출력**
- 각 차량 PDF 다운로드: `GET /linehaul/slip/generate?truckRequestId={ID}`
- 🚨 **인쇄 순서: 3호차 → 2호차 → 1호차** (거꾸로)
  - 이유: 1호차가 맨 위로 정렬되어 바로 사용 가능
- 무음 인쇄: `os.startfile(pdf_path, 'print')`
- 장당 간격: `time.sleep(3.0)`

**Phase 2: KPP PBM140MW 출하통보 등록**
- 기사명/연락처 확보 후 KPP 로그인 → PBM140MW 등록
- 🚨 **11톤(3호차) 기본 수량 = 14개** (5T=12개, 11T=14개)
- 통보 형식: `[KPP 팔레트 등록 완료 - {날짜}]`

### 2.2 15:00 — LS 등록 재확인 (Watchdog)

- LS VF67_H 조회
- ✅ 차량 등록됨 → **`[SILENT]`** (메시지 없음)
- 🚨 차량 미등록 (0건) → **알람 전송**: "다른 쪽에 요청해 주세요"
- ⚠️ 쿠키 만료 → **"확인 불가 (13시 등록은 정상)"** 메시지 (오탐지 방지)

### 2.3 16:00 — LS 등록 2차 재확인 (Watchdog)

- LS VF67_H 조회 (15시와 동일 로직)
- ✅ 차량 등록됨 → **`[SILENT]`**
- 🚨 차량 미등록 → **2차 알람**: "여전히 등록되지 않음"

## 5. 규칙 요약

### 🚨 절대 규칙

| 규칙 | 설명 |
|:-----|:------|
| **Watchdog = 재등록 금지** | 15시/16시는 확인만. 절대 차량 등록/수정 금지 |
| **11T 팔레트 = 14개** | 3호차(11톤) 기본 등록 수량은 14개. 12개 금지 |
| **인쇄 순서 = 3→2→1** | 거꾸로 인쇄해야 1호차가 맨 위로 정렬됨 |
| **Phase 1 실패 = 중단** | LS 실패 시 PDF/KPP 진행하지 않음 |
| **등록됨 = SILENT** | 15시/16시 크론: 등록되어 있으면 무조건 조용히 |

### ⚠️ 주의사항

| 항목 | 내용 |
|:-----|:------|
| **LS 필터** | `locationStart=VF67_H` 필수 (VF67(코드만) → 0건) |
| **브라우저 쿠키** | CDP Network.getCookies 자동 추출 우선, 실패 시 수동 재발급 |
| **쿠키 만료** | WEB-GATEWAY-SESSION 약 1~2시간 → 15시/16시는 만료 가능성 높음 |
| **호차** | 등록 순서 (시간 아님) |
| **오늘 날짜** | 일자 묻지 말고 자동 계산 |
| **에러** | 500/에러 시 1회 재시도 후 실패 보고 |
