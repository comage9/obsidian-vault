## 재구축 절차

### Step 1: 기존 DB 백업
- `ls_vehicle_db_full.json` → `ls_vehicle_db_full.json.bak.20260610`

### Step 2: CDP Browser Fetch로 Tracking API 주간 조회

- **23주 분량** (2026-01-01~2026-06-10, 일주일 단위)

### Step 3: PDF 다운로드
- 매 레코드마다 `GET /linehaul/slip/generate?truckRequestId={id}` CDP fetch
- `FileReader.readAsDataURL(blob)` → base64 디코드 → 로컬 저장
- 총 **484건 PDF 다운로드** (약 1.5MB/건)

### Step 4: 텍스트 추출 (PyMuPDF)
- 초기 `pdftotext` 실행 실패 (git-bash PATH ≠ Windows PATH)
- → **PyMuPDF(`import fitz`)** 로 전환하여 PDF 텍스트 추출 성공
- 최근 7일(6/4~6/10) 21건 드라이버 정보 추출 완료
- 6/4 이전 PDF는 LS 정책에 따라 `[DELETED]` 처리됨

### Step 5: DB 구조 생성

- `vehicle_db_merged.json`: VF 대시보드용 간소화 버전 (69개 차량-기사 결합)

doc = fitz.open(stream=pdf_data, filetype='pdf')
text = ''

doc.close()
