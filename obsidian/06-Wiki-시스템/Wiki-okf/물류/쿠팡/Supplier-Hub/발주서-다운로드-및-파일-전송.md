## 부록 A: EUC-KR ZIP 파일명 처리 절차 (2026-06-07 추가)

### 문제
ZIP 내부 xlsx 파일명이 EUC-KR(`ë°ì£¼ìë¦¬ì¤í¸_*.xlsx`)로 인코딩되어 있음.
`unzip`으로 풀면 파일명이 깨져 보임.

### 해결 절차
```bash
# 1. ZIP 다운로드 (ASCII 이름으로 저장)
curl -s -o /tmp/po_original.zip \
  -H "Cookie: $COOKIES" \
  "https://supplier.coupang.com/scm/purchase/order/excelList?purchaseOrderSeqList=132952001,132952067"

# 2. ZIP 풀기 (파일명 깨짐 무시)
mkdir -p /tmp/po_xlsx && cd /tmp/po_xlsx
unzip -o /tmp/po_original.zip 2>/dev/null

# 3. 깨진 파일명 → ASCII로 복사
for f in *.xlsx; do
  po=$(python3 -c "import re; print(re.search(r'\d{9}', '$f').group())" 2>/dev/null)
  [ -n "$po" ] && cp "$f" "/tmp/po_${po}.xlsx"
done
```

**절대 깨진 파일명을 그대로 텔레그램 전송 금지.**

## 1. 발주서 다운로드 API

### 1-1. 단일 PO 다운로드 (xlsx)

```
GET https://supplier.coupang.com/scm/purchase/order/excel
  ?purchaseOrderSeq={PO번호}

응답: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
      Content-Disposition: attachment; filename="PO_{번호}.xlsx"
      Body: ~20KB xlsx 바이너리
```

### 1-2. 다중 PO 다운로드 (zip 묶음, 서버 기본 동작)

```
GET https://supplier.coupang.com/scm/purchase/order/excelList
  ?purchaseOrderSeqList={PO1},{PO2},...

응답: application/zip
      Content-Disposition: attachment; filename="발주서_YYYYMMDDhhmmss.zip"
      Body: ZIP 묶음 (각 PO별 .xlsx, EUC-KR 파일명)
```

**주의**: 쿠팡 서버는 다중 선택 시 **무조건 ZIP 묶음**으로 응답. JS 소스 `poList.length === 1` 분기에서 단일일 때만 xlsx endpoint 호출. 통합 xlsx endpoint 없음.

### 1-3. 다중 PO → xlsx 통합 절차 (수동)

ZIP 풀기 → openpyxl로 5개 xlsx 읽기 → 입고예정일 기준 필터 → 통합 xlsx 생성:

```python
# /tmp/po_xlsx/ 에 ZIP 풀린 5개 xlsx 있다고 가정
import openpyxl, glob, os
from openpyxl import Workbook

target = {'132952001', '132952067', '132952072'}  # 6/2 입고예정일
out = Workbook(); out.remove(out.active)
for fp in sorted(glob.glob('/tmp/po_xlsx/*.xlsx')):
    po = os.path.basename(fp).split('_')[-1].replace('.xlsx','')
    if po in target:
        src = openpyxl.load_workbook(fp, data_only=True)
        out.create_sheet(f'PO_{po}')  # 시트 단위로 통합
        for row in src.active.iter_rows(values_only=True):
            out.active.append(row)
out.save('/tmp/po_6월2일_supplier.xlsx')
```

**입고예정일 위치 (R12/R13, F열)**:
- R12: `입고예정일시 | 물류센터 | 주소 | | 입고예정일시 | 하차일시`
- R13: ` | | 유원피에스 | 경기도 양주시... | | 2026/06/02 00:00:00 | ...`

## 2. 텔레그램 채팅방에 파일 첨부 전송 (실패 경험 정리)

### 2-1. ❌ 안 되는 방법

| 방법 | 실패 이유 |
|---|---|
| `send_message` 툴 + `MEDIA:/tmp/...` (한글 경로) | 봇 어댑터 `MEDIA_TAG_CLEANUP_RE`가 확장자 매칭 실패 → 텍스트로만 처리 |
| `send_message` 툴 + `MEDIA:` 마커 미사용 | 봇 어댑터가 텍스트만 전송, 파일 추출 안 함 |
| `curl`로 Telegram `sendDocument` API 직접 호출 | 게이트웨이 어댑터 우회 → 봇이 보내긴 하나 게이트웨이가 이 채팅방 알림에 반영 안 함 (사용자 화면에 안 보임) |
| 빈 텍스트 + `MEDIA:` 만 | "Tool result missing" / 봇 어댑터 자체 죽음 |

### 2-2. ✅ 되는 방법 (검증됨, 2026-06-03)

**`send_message` 툴 + ASCII 경로 + 짧은 텍스트**:

```python
send_message(
    message="6/2 입고예정 PO 3건 통합 파일 첨부합니다.\n\n"
            "MEDIA:/tmp/po_6월2일_supplier.xlsx\n"
            "MEDIA:/tmp/po_132952001.xlsx\n"
            "MEDIA:/tmp/po_132952067.xlsx\n"
            "MEDIA:/tmp/po_132952072.xlsx",
    target="telegram"  # 또는 chat_id=5708696961
)
```

**응답** (게이트웨이 로그):
- `success: true`
- `chat_id: "5708696961"`
- `message_id: 883`
- `note: "Sent to telegram home channel (chat_id: 5708696961)"`

### 2-3. 핵심 규칙

1. **파일 경로는 ASCII만** (한글/공백/특수문자 → 파싱 실패)
2. **파일명 끝은 deliverable 확장자** (.xlsx, .zip, .pdf, .png, .jpg 등)
3. **경로는 절대경로** (`/tmp/...`)
4. **메시지 본문은 짧게** (긴 본문 + 파일 혼합 시 어댑터 문제 발생 가능)
5. **파일은 `cp`로 ASCII 이름 복사본 만들어서 보내기** (원본은 보존)

```bash
# 한글 파일명 → ASCII 복사
cp /tmp/ë°ì£¼ìë¦¬ì¤í¸_*.zip /tmp/po_original_5.zip
cp /tmp/po_xlsx/*132952001.xlsx /tmp/po_132952001.xlsx
```

### 2-4. 봇 어댑터 처리 로직 (참고)

게이트웨이 `gateway/platforms/base.py` L3783~L3797:

```python
# Extract MEDIA:<path> tags (from TTS tool) before other processing
media_files, response = self.extract_media(response)
media_files = self.filter_media_delivery_paths(media_files)

# Strip only MEDIA: tags whose path has a deliverable extension
text_content = MEDIA_TAG_CLEANUP_RE.sub("", text_content).strip()
```

정규식 `MEDIA_TAG_CLEANUP_RE` (L1200~L1206):
- `MEDIA:` 마커 + 절대경로/홈상대경로 + **알려진 확장자** (.xlsx 등) 매칭
- 확장자 모르는 경로는 **텍스트로 남김** (조용히 누락 방지)

## 3. 쿠키 추출 & 로그인

`browser-cookie3` 로 너 PC Chrome `supplier.coupang.com` 쿠키 추출 → JSON → Netscape 변환:

```python
import browser_cookie3
import json
cj = browser_cookie3.chrome(domain_name='supplier.coupang.com')
data = [{'name': c.name, 'value': c.value, 'domain': c.domain, 'path': c.path} for c in cj]
with open('/tmp/coupang_supplier_cookies.json', 'w') as f:
    json.dump(data, f, indent=2)
```

**Netscape 변환** 후 `python-requests` Session에 주입:
- `cp001`, `JSESSIONID`, `SID`, `XSRF-TOKEN` 등 4~6개
- `requests.get(..., cookies=netscape_dict)` → 로그인 상태로 발주 조회 가능
