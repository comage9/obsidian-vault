# VF 바코드 시스템 ↔ VF 프로젝트 통합 계획 v2

## 1. 핵심 목적

**VF 바코드 = 입력 데이터를 바코드로 생성하는 메인 인터페이스**

생성하는 바코드 2종:
1. **입력 데이터 기반 바코드** — 사용자가 입력한 출고 데이터를 바코드로 생성
2. **송장 번호 바코드** — 기존 송장 번호를 바코드로 생성

## 2. 데이터 흐름 (목표)

```
┌─────────────────────────────────────────────────────┐
│                   공유 DB (단일 소스)                   │
│  ┌─────────────────┐  ┌───────────────────────────┐  │
│  │ VF 프로젝트 데이터 │  │ VF 바코드 입력 데이터 (추가) │  │
│  │ (기존 품목/수량)   │  │ (신규 출고 데이터)          │  │
│  └─────────────────┘  └───────────────────────────┘  │
│           ↓                      ↓                    │
│  ┌─────────────────────────────────────────────┐     │
│  │         최종 출고 수량 = 합산                 │     │
│  └─────────────────────────────────────────────┘     │
└─────────────────────────────────────────────────────┘
         ↕ (동기화)
┌──────────────────────┐
│   Google Sheets       │
│   (백업 용도 유지)      │
└──────────────────────┘
```

### 핵심 원칙
1. **기존 데이터는 DB에 보존** — VF 프로젝트의 기존 품목/수량 데이터는 유지
2. **신규 데이터는 추가 입력** — VF 바코드 페이지에서 입력되는 데이터는 기존 DB에 **추가** (덮어쓰기 X)
3. **최종 출고 수량 = 합산 결과** — 기존 데이터 + 신규 입력 데이터의 합계
4. **Google Sheets는 백업 용도** — 완전 연결 해제 X, 동기화 유지
5. **같은 데이터 공유** — VF 프로젝트와 VF 바코드가 동일 DB 조회

## 3. 통합 계획 (3단계)

### Phase A: React 컴포넌트화 ⏱ 2-3시간

**목표:** 독립 HTML → React 컴포넌트로 전환

| 작업 | 내용 | 상세 |
|:-----|:-----|:-----|
| A-1 | `pages/barcode-generation.tsx` 생성 | 기존 `index.html`의 HTML → JSX, CSS → Tailwind |
| A-2 | 상태 관리 통합 | `useDeliveryData` 훅 → 대시보드 API 재사용 |
| A-3 | 사이드바 교체 | `externalUrl` → `path: "/barcode"` + React Router |
| A-4 | 제품 DB 통합 | 바코드 페이지에서 VF 프로젝트 DB 조회 (동일 소스) |

### Phase B: 데이터 통합 ⏱ 2-3시간

**목표:** 기존 DB + 신규 입력 데이터 통합 관리

| 작업 | 내용 | 상세 |
|:-----|:-----|:-----|
| B-1 | **DB 스키마 확장** | 기존 VF 프로젝트 테이블에 신규 출고 데이터 컬럼 추가 (또는 연결 테이블) |
| B-2 | **데이터 추가 API** | `POST /api/delivery/hourly` — 신규 데이터만 추가 (기존 데이터 수정 X) |
| B-3 | **합산 조회 API** | `GET /api/delivery/total?date=YYYY-MM-DD` — 기존 + 신규 합산 결과 반환 |
| B-4 | **Google Sheets 동기화 유지** | DB 변경 시 Sheets에도 반영 (백업 유지, 단방향 또는 양방향) |

### Phase C: 기능 고도화 ⏱ 선택

| 작업 | 내용 |
|:-----|:-----|
| C-1 | 실시간 바코드 스캔 연동 (웹캠/모바일) |
| C-2 | 인쇄 템플릿 커스터마이징 |
| C-3 | 출고 데이터 → 생산 계획 자동 반영 |

## 4. 데이터 모델 (제안)

```sql
-- 기존 VF 프로젝트 테이블 (유지)
CREATE TABLE vf_delivery_items (
    id INTEGER PRIMARY KEY,
    product_code TEXT,
    product_name TEXT,
    category TEXT,
    quantity INTEGER,
    delivery_date DATE,
    time_slot TEXT,
    created_at DATETIME
);

-- 신규 바코드 입력 테이블 (추가)
CREATE TABLE barcode_input_items (
    id INTEGER PRIMARY KEY,
    product_code TEXT,
    product_name TEXT,
    category TEXT,
    quantity INTEGER,
    delivery_date DATE,
    time_slot TEXT,
    barcode_data TEXT,
    tracking_no TEXT,
    source TEXT DEFAULT 'barcode',
    created_at DATETIME
);

-- 통합 조회 뷰
CREATE VIEW vw_daily_delivery_total AS
SELECT 
    delivery_date,
    product_code,
    product_name,
    category,
    SUM(quantity) as total_quantity
FROM (
    SELECT delivery_date, product_code, product_name, category, quantity 
    FROM vf_delivery_items
    UNION ALL
    SELECT delivery_date, product_code, product_name, category, quantity 
    FROM barcode_input_items
) GROUP BY delivery_date, product_code;
```
