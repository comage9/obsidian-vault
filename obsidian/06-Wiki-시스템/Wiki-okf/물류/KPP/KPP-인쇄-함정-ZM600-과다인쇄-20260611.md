# KPP 인쇄 함정 — ZM600 기본프린터 + 과다인쇄 방지

## 개요

2026-06-11 실제 사고를 기반으로 KPP/LS PDF 인쇄 시 반드시 알아야 할 2가지 함정을 정리.

---

## 함정 1: ZM600 기본프린터 — Canon G2010 series 명시 필수

### 문제
Windows 기본 프린터가 `ZDesigner ZM600` 라벨 프린터로 설정되어 있음.
`os.startfile(pdf_path, 'print')`는 **기본 프린터**로 인쇄하므로,
일반 PDF 문서가 ZM600 라벨 프린터로 전송되어 **인쇄 실패/출력물 무용지물** 발생.

### 사고 사례
- 2026-06-11: `os.startfile(pdf, 'print')` 호출 → ZM600 라벨 프린터로 일반 PDF 전송
- 라벨 프린터가 일반 A4 문서를 인쇄하지 못하거나, 소모성 라벨지에 일반 내용 출력
- 사용자: "왜 인쇄가 안 됐지?"

### 해결

| 구분 | 방법 | 설명 |
|:----|:-----|:-----|
| ❌ **금지** | `os.startfile(pdf, 'print')` | 기본 프린터(ZM600) 사용 |
| ❌ **금지** | `win32print.SetDefaultPrinter()` 변경 | 타 앱(ZM600 정상 사용) 영향 |
| ✅ **권장** | `win32api.ShellExecute(0, "printto", pdf, '"Canon G2010 series"', ".", 0)` | 프린터 명시 지정 |

### 코드 예시
```python
import win32api

def print_pdf_canon(pdf_path):
    """Canon G2010 series로 명시적 인쇄 (ZM600 우회)"""
    from win32api import ShellExecute
    ShellExecute(0, "printto", pdf_path, '"Canon G2010 series"', ".", 0)
```

### 검증
```python
import win32print
default = win32print.GetDefaultPrinter()
print(f"기본 프린터: {default}")  # ZM600이면 주의
```

## 함정 2: 과다인쇄 — 1회만 전송

### 문제 (실제 사고, 2026-06-11)
인쇄 명령이 중복 전송되어 **70장 이상** 과다 출력.
- 프린터 용지/토너 낭비
- 출력물 정리에 불필요한 시간 소요
- 사용자 극도 불편

### 원인
1. 루프 내에서 `os.startfile` 호출 → 매 반복마다 인쇄
2. 여러 인쇄 방식 순차 실행 → 동일 PDF가 중복 출력
3. 실패 시 자동 재시도 로직 → 연쇄 인쇄

### 해결
| # | 규칙 | 상세 |
|---|------|------|
| 1 | **인쇄 전 대수 확인** | `print(f"인쇄 전송: {N}대 × 1장")` 로그 출력 필수 |
| 2 | **1회만 전송** | `os.startfile`는 정확히 1회, 건수 분리 |
| 3 | **실패 시 재시도 금지** | → "인쇄 실패, 수동 확인 필요" 사용자 보고 |
| 4 | **4초 간격 유지** | 연속 PDF 인쇄 시 Acrobat 충돌 방지 |

### 올바른 패턴
```python
hoches = [1, 2, 3]
print(f"인쇄 전송: {len(hoches)}대 × 1장")

for hoche in hoches:
    pdf = f"C:/temp/kpp_edi_{hoche}hoca.pdf"
    ShellExecute(0, "printto", pdf, '"Canon G2010 series"', ".", 0)
    time.sleep(4)  # 연속 인쇄 충돌 방지

print("인쇄 완료 (1회 전송)")
```

### 잘못된 패턴 (금지)
```python
# ❌ 루프 내부에서 os.startfile 중복 호출
for hoche in hoches:
    os.startfile(pdf, 'print')  # ← 매번 전송 → 3장씩 N회
```
