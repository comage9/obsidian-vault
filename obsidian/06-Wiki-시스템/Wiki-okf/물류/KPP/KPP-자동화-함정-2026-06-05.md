# KPP 자동화 함정 (2026-06-05 발견)

## 1. React Controlled Input 패턴

**문제:** `input.value = '2026-06-05'` 후 search 클릭 시 React state가 업데이트 안 됨 → 옛날 조건으로 조회됨.

**해결:**
```javascript
var nativeInputValueSetter = Object.getOwnPropertyDescriptor(
    iw.HTMLInputElement.prototype, 'value'
).set;
nativeInputValueSetter.call(fld, '2026-06-05');
fld.dispatchEvent(new Event('input', { bubbles: true }));
fld.dispatchEvent(new Event('change', { bubbles: true }));
```

**검증:** 6/3 데이터 보이던 게 6/5 setValue 후 진짜 6/5로 조회됨.

**대상 필드:**
- PBM140MW: `sr_dlv_dat_f`, `sr_dlv_dat_t`
- PBM110MW: `REQ_DAT_FR`, `REQ_DAT_TO`

## 2. Native Alert 자동 닫기

**문제:** WPPS 메시지 팝업(예: "승인거부 데이터")은 native `window.alert` — `gfn_alert` override로 못 잡음.

**해결:**
```python
# Page.handleJavaScriptDialog (dialog 떠있을 때만)
cmd('Page.handleJavaScriptDialog', {'accept': True})

# 또는 백그라운드 핸들러로 미리 등록
import threading, json, websocket
def dialog_handler():
    ws2 = websocket.create_connection(page['webSocketDebuggerUrl'])
    while True:
        try:
            msg = json.loads(ws2.recv())
            if msg.get('method') == 'Page.javascriptDialogOpening':
                ws2.send(json.dumps({'id': 999, 'method': 'Page.handleJavaScriptDialog', 'params': {'accept': True}}))
        except: break

t = threading.Thread(target=dialog_handler, daemon=True)
t.start()
```

**WPPS 메시지 팝업 내용 예시:**
```
[승인거부 데이터]
2026-06-03 / 610060 쿠팡-부천1센터[HUB] / 승인거부사유: 제품 미입고
2026-06-03 / 610060 쿠팡-부천1센터[HUB] / 승인거부사유: 제품 미입고
```

## 3. Edge sync-confirmation-dialog (1회성)

**문제:** Edge 첫 시작 시 `edge://sync-confirmation-dialog/` 탭이 떠서 Microsoft 계정 동기화 동의 요청. `cr-button` shadow root 깊숙한 곳에 확인 버튼 있어 일반 `querySelector`로 못 찾음.

**해결:** 사용자가 [확인] 1회 눌러주면 이후부터 안 뜸. 또는 `--user-data-dir=new`로 새 프로필로 시작.
