### ⚠️ dialog 처리 (가장 중요! 반드시 숙지)
1. WPPS는 `window.alert()` native dialog 사용 (조회 시 "승인거부 데이터", 저장 시 "저장되었습니다")
2. dialog이 뜨면 해당 페이지의 **모든 CDP WebSocket 연결이 차단됨**
3. `Runtime.evaluate` 명령은 dialog 때문에 전부 타임아웃
4. **해결 순서:**
   a. `window.alert = function(){}` override (gfn_alert만 override로 부족)
   b. background thread로 `Page.javascriptDialogOpening` 이벤트 감지 → `Page.handleJavaScriptDialog` 호출
   c. 실패 시: browser-level WebSocket으로 `Input.dispatchKeyEvent` Enter 전송
   d. 그래도 안 되면: blocked page close + 새 페이지 열고 재시도

### 표준 코드 패턴
```python
# STEP 1: dialog override (필수!)
cmd('Runtime.evaluate', {'expression': '''
    window.alert = function(msg){ console.log(msg); };
    window.confirm = function(msg){ return true; };
    gfn_alert = function(msg){ return msg; };
    gfn_confirm = function(msg){ return true; };
'''})

# STEP 2: background dialog handler
def dialog_handler(ws_url):
    dws = websocket.create_connection(ws_url, timeout=10)
    dws.settimeout(60)
    while True:
        try:
            msg = json.loads(dws.recv())
            if msg.get('method') == 'Page.javascriptDialogOpening':
                dws.send(json.dumps({'id':999,
                    'method':'Page.handleJavaScriptDialog',
                    'params':{'accept':True}}))
        except: break

threading.Thread(target=dialog_handler, args=(current_ws_url,), daemon=True).start()

# STEP 3: action 실행
cmd('Runtime.evaluate', {'expression': 'document.getElementById("search").click()'})
time.sleep(5)

# STEP 4: force accept (fallback)
try:
    fws = websocket.create_connection(ws_url, timeout=3)
    fws.send(json.dumps({'id':1,'method':'Page.handleJavaScriptDialog','params':{'accept':True}}))
    fws.close()
except: pass
```
