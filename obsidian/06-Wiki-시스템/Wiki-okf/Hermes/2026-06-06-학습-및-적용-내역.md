## 2. 코드 수정 내역 (VF 출고 바코드 시스템)

> 대상 파일: `E:\coding\VF 출고 바코드\vf 출고 바코드 생성 (1).html`

### 2-1. 시간 체크 버그 수정 (HIGH)

**문제:** `for (let h = 0; h <= now; h++)` — 현재 시각(20시)을 포함하여 20시도 미입력으로 체크
**수정:** `for (let h = 0; h < now; h++)` — 현재 시각 미포함
**결과:** 20:09에 19시까지 입력 완료 = 통과 ✅

### 2-2. 전송값 계산 오류 수정 (HIGH)

**문제:** `lastCumul += val` — 사용자 입력값 250을 이전 누적(210)에 더해서 460 전송
**수정:** `entries.push({ hour, quantity: val })` — 입력값 그대로 전송
**결과:** 입력 250 = API 저장 250 ✅

### 2-3. parseInt radix + isNaN 체크 추가 (MEDIUM)

**문제:** `parseInt(inp.dataset.hour)` — radix 없이 8진수 오해 가능
**수정:**
```javascript
const hour = parseInt(inp.dataset.hour, 10);
const val = parseInt(inp.value, 10);
if (!isNaN(hour) && !isNaN(val) && val > 0) {
    entries.push({ hour, quantity: val });
}
```
**결과:** 10진수 강제 + hour NaN 방지 ✅

## 3. 하네스 교차 검증 상세 (핵심 방법론)

### 3-1. 검증 체인

```
M3 (MiniMax) — 메인 검토
    ├── 불일치 시 → DS-V4-flash — 1차 교차 검증
    │                   └── 불일치 시 → nemotron(OpenRouter 무료) — 2차 교차 검증
    └── 일치 시 → 3차 생략 (시간 절약)
```

### 3-2. 호출 패턴

```python
# DS-V4-flash (검증 1차)
url: https://api.deepseek.com/v1/chat/completions
model: deepseek-v4-flash
API key: DEEPSEEK_API_KEY (.env)

# nemotron (검증 2차/3차) — OpenRouter 무료
url: https://openrouter.ai/api/v1/chat/completions
model: nvidia/nemotron-3-ultra-550b-a55b:free
API key: OPENROUTER_API_KEY (.env, sk-or-v1- 형식)
참고: nemotron은 한글 처리 안 됨 → **영문 프롬프트** 필수
```

### 3-3. 이번 검증 결과

| 모델 | 응답 | 소요시간 |
|------|------|----------|
| M3 (메인) | 정상 | 즉시 |
| DS-V4-flash (검증1) | 빈 응답 (2회) | — |
| nemotron (검증2) | 버그 4개 식별 | ~51s |

**발견된 버그:**
1. 시간 체크 `h <= now` → `h < now` (HIGH)
2. 전송값 `lastCumul += val` → `val` 직접 (HIGH)
3. `parseInt()` radix 누락 (MEDIUM)
4. `isNaN(hour)` 체크 누락 (MEDIUM)

### 3-4. Pitfalls (중요)

1. **DS-V4-flash 간헐적 빈 응답** — HTTP 200이지만 content="" 반환. 재시도 필요
2. **nemotron 한글 불가** — 영문 프롬프트 필수 (mojibake 발생)
3. **타임아웃** — M3/DS: 30s, nemotron: **90s** (실측 51s~91s)
4. **OpenRouter 키 형식** — `sk-or-v1-...` 표준 형식 확인
5. **3차 생략 규칙** — M3+DS 일치 시 nemotron 생략 (시간 절약)

## 7. 이번 작업에서 배운 점

1. **"했다"고 말하기 전에 실제 검증** — API 리셋 후 반드시 GET 조회로 확인
2. **사용자 요청 = 단순하게** — "계산하지마, 값 그대로" = `lastCumul += val` 제거
3. **경계 조건 항상 체크** — `h <= now` vs `h < now` (현재 시각 포함 여부)
4. **하네스 교차 검증 = 3모델** — 각각의 특성 파악 필수 (DS 빈응답, nemotron 한글불가)
