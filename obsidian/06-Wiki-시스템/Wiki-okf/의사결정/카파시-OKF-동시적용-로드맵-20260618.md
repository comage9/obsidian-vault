```yaml
name: karpathy-4-principles
description: 카파시 코딩 4원칙 운영원칙 3 flavor — D8(Think)·D7(Simple)·INV(Surgical)·NOTICE(Goal-oriented). 결정고정(§7) 최상위.

---
```

**3 Flavor 정의** (~1.5KB):

```markdown
> **이 스킬은 결정고정(§7) 아래에 있다.**
> 사용자가 "네/진행/OK" 등으로 1회 확인 = 즉시 실행.
> D8(Think)은 사고 강제이지 사용자 재질문이 아니다.

## Flavor D8 — "Think" (사고 강제)
- 목표·제약·입출력 먼저 명시
- 위반: "추측으로 시작" / "일단 해보고 생각"
- 준수: "목표=X, 제약=Y, 입출력=Z"

## Flavor D7 — "Simple" (단순성 강제)
- 기존 도구/스킬 우선. 새로 만들기 전 검증.
- 위반: "기존 스킬 무시하고 새로 만들기"
- 준수: "skills_list() 확인 → 50%+ 커버 시 기존 사용"

## Flavor INV — "Surgical" (정밀성 강제)
- 변경은 필요한 부분만
- 위반: "git checkout으로 사용자 변경사항 소실" / "원본 그대로 지시 + 임의 수정"
- 준수: "git status 먼저 → stash 백업 → 작업"

## Flavor NOTICE — "Goal-oriented" (목표지향)
- 모든 작업은 3중 완료 (Wiki + README + Git push)
- 위반: "됐다고 보고 + 검증 생략"
- 준수: "API GET → 실제 값 인용 → Wiki 기록 → README → Git push"
```
