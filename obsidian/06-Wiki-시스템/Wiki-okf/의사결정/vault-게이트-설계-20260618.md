## Vault Gate가 뭔가 (쉽게)

### 비유
Hermes가 "내 위키(지식창고)가 어디 있지?" 찾을 때 3단계로 확인하는 안전장치:

```
[1순위] "오늘 출근할 때 어디로 가라고 했지?"  → 환경변수(WIKI_PATH)
   ↓ 없거나 경로가 없음
[2순위] "그럼 명찰에 적힌 회사 주소는?"  → 포인터 파일(~/.wiki_location)
   ↓ 없거나 파일이 없음
[3순위] "본사 주소로 가자"  → 기본값(~/wiki 또는 OS별 기본 경로)
```

### 왜 필요한가 (배경)

**현재 문제 2가지:**

| 상황 | 문제 |
|------|------|
| ① GUI 환경 (Telegram/Discord/Desktop) | 환경변수 설정이 안 먹힘. "WIKI_PATH 없음" 에러 |
| ② 여러 기기 (Windows ↔ Linux) | 각 기기마다 env를 따로 설정. 한쪽 빠지면 작동 안 함 |

**knot 영상 패턴:**
- netwaif/multi-agent-starter 커밋 메시지:
  > `fix(knot): vault 게이트 env→포인터파일 fallback — GUI 호스트앱 진입장벽 제거`
- 즉 knot도 같은 문제를 겪었고, env → 포인터파일 fallback으로 해결

## WIKI_PATH 환경변수란

Hermes의 위키(지식창고) 위치를 가리키는 환경변수:

```bash
# 터미널에서 설정
WIKI_PATH=E:\hermes-backup\obsidian\06-Wiki-시스템\Wiki
```

**현재 Hermes 동작:**
- `research/llm-wiki/SKILL.md`의 "Wiki Location" 섹션에 정의됨
- ```bash WIKI="${WIKI_PATH:-$HOME/wiki}" ```
- env가 없으면 `~/wiki` 기본값 사용

---

## 포인터 파일이란

```
파일 위치: ~/.wiki_location  (홈 디렉토리/.wiki_location)
파일 내용: E:\hermes-backup\obsidian\06-Wiki-시스템\Wiki
            ↑ 한 줄짜리 텍스트. 위키가 어디 있는지 적혀 있음.
```

**장점:**
- 환경변수 설정 없이도 작동 (GUI 환경 OK)
- 한 번만 적어두면 모든 Hermes 세션이 자동으로 같은 위키 사용
- 여러 기기에서 Syncthing 등으로 포인터 파일만 동기화하면 끝

**단점:**
- 임시로 다른 위키 쓰고 싶을 때 env 바꿔도 포인터가 있으면 env 무시될 수 있음
- → **순서 중요: env가 항상 1순위**

## Vault Gate 구현 설계

### Python 스크립트 (Phase 2 산출물)

**경로:** `research/llm-wiki/scripts/vault_gate.py`

```python
"""
Hermes Wiki Vault Gate
=======================
Knot 패턴(env→포인터파일 fallback)을 Hermes llm-wiki에 이식.

우선순위:
  1. WIKI_PATH 환경변수 (있고 경로가 존재하면)
  2. ~/.wiki_location 포인터 파일 (있고 경로가 존재하면)
  3. OS별 기본값 (Windows: %USERPROFILE%\wiki, Linux/Mac: ~/wiki)
"""
import os
from pathlib import Path


def resolve_wiki_path() -> Path:
    """
    Wiki vault 경로를 3단계 fallback으로 결정.
    
    Returns:
        Path: 실제 사용 가능한 wiki 경로
    
    Raises:
        FileNotFoundError: 3단계 모두 실패 시
    """
    # 1순위: 환경변수
    env_path = os.environ.get("WIKI_PATH")
    if env_path:
        p = Path(env_path)
        if p.exists() and p.is_dir():
            return p.resolve()
    
    # 2순위: 포인터 파일
    pointer = Path.home() / ".wiki_location"
    if pointer.exists() and pointer.is_file():
        try:
            ptr_path = Path(pointer.read_text(encoding="utf-8").strip())
            if ptr_path.exists() and ptr_path.is_dir():
                return ptr_path.resolve()
        except (OSError, ValueError):
            pass  # 파일 읽기 실패 → 다음으로
    
    # 3순위: OS별 기본값
    default = Path.home() / "wiki"
    if default.exists() and default.is_dir():
        return default.resolve()
    
    # 3단계 모두 실패
    raise FileNotFoundError(
        "Wiki vault를 찾을 수 없습니다. WIKI_PATH 환경변수 또는 "
        "~/.wiki_location 포인터 파일, 또는 ~/wiki 폴더 중 하나가 필요합니다."
    )


if __name__ == "__main__":
    wiki = resolve_wiki_path()
    print(f"Wiki vault: {wiki}")
```

## 발생할 수 있는 문제 (3개)

### P1 — env 임시 변경 무시

**시나리오:** 사용자가 임시로 다른 wiki 작업하려고 `export WIKI_PATH=...` 설정했는데 포인터 파일이 있으면 env 무시.

**대응:** 1순위는 항상 env. 포인터 파일은 명시적으로 무시할 방법 없음 (의도된 동작).

### P2 — 포인터 파일 경로 동기화 충돌

**시나리오:** Windows와 Linux에서 Syncthing 동기화 시 포인터 파일 경로가 OS별로 다름.

**대응:**
- Windows: `E:\hermes-backup\obsidian\06-Wiki-시스템\Wiki\`
- Linux: `/media/comage/data1/hermes-backup/obsidian/06-Wiki-시스템/Wiki/`
- 동기화 시 `.gitignore`에 `~/.wiki_location` 추가하거나 OS별 포인터 분리

### P3 — 권한 오류

**시나리오:** `~/.wiki_location` 파일이 root 소유이거나 권한 없음.

**대응:** `try/except OSError`로 처리 (이미 설계에 포함됨).
