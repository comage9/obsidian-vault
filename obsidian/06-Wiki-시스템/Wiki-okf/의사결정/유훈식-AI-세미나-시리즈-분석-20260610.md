# 유훈식 AI&UX 세미나 시리즈 분석 (2026-06-10)

## 1. [63회] 옵시디언과 헤르메스에게 UX 디자인 시켜보자
**발표자:** 유훈식 교수 (AI4UX, 구독자 1.16만)
**길이:** 1시간 3분

### 핵심 내용
1. **AI 네이티브 조직** — Tool→Assistant→Teammate→System 4단계 전환
2. **3대 자유령 에이전트** — Hermes Agent(자기진화/메모리중심), OpenCodex(멀티에이전트), Gemini Spark(Google IO 2026)
3. **LLM Wiki = 카파시 개념** — Wiki(데이터) + LM(모델) + IDE(Obsidian)
4. **실제 데모** — 한 줄 명령 → 인터뷰계획→인터뷰→분석→페르소나→PRD→HTML디자인→사용성평가 전과정 자동화
5. **마크다운 효율성** — HTML 대비 토큰 65~90% 감소

### 우리 시스템에 적용
- 100% 동일 아키텍처 (Hermes + Obsidian + LLM Wiki)
- 50+ 스킬 (유훈식 교수 35개 UX 스킬보다 더 많음)
- Telegram 연동, 크론 자동화, 메모리 기반 자기 진화 모두 동일

## 2. UX/UI 디자이너에게 옵시디언이 필수가 되고 있는 이유
**길이:** 39분 | **조회수:** 3,700회

### 핵심 내용
1. **LM Wiki 패러다임** — 안드레 카파시가 제시한 LLM + Wiki 구조
2. **Obsidian = AI 인터페이스** — 마크다운 기반, 연결성(노드) 구조가 LLM 친화적
3. **토큰 효율성** — HTML 대비 30~40% 절감
4. **스킬화** — 모든 UX/UI 디자인 지식을 마크다운 스킬로 자산화
5. **6모듈 AI 네이티브 교육**: LM Wiki → Vibe Research → Vibe Design → GI Source Dev → Vibe Coding → Agentic Engineering

### 적용
- 스킬 체계화(trigger_condition + output_template) 진행 완료
- AI-DTQ 자격증 과정 참고

## 3. 2026 LLM 실무 종합 — sLLM·GraphRAG·환각해결
**발표자:** LG전자 AI 연구소 요한 박사 외 2인
**길이:** 4시간 | **조회수:** 3,200회

### 핵심 내용
1. **sLLM 파인튜닝** — 양자화(Quantization), PEFT(LoRA), 로컬 환경에서 SLM 학습
2. **GraphRAG** — Knowledge Graph + Vector Search 하이브리드로 환각 해결
3. **Knowledge Graph 구축** — PDF/논문에서 엔티티-관계 추출, Cypher 검색
4. **Tool 라우팅** — LLM이 질문 도메인에 맞는 검색 도구 자동 선택

### 우리 시스템 적용 가능
1. **GraphRAG 도입 검토** — Obsidian Wiki의 노드 연결 구조를 Knowledge Graph로 활용
2. **벡터 검색 + 그래프 탐색 하이브리드** — Obsidian 링크 + 임베딩 검색 결합

## 결론
우리 시스템(Hermes + Obsidian + LLM Wiki)은 유훈식 교수가 제시한 AI 네이티브 UX 시스템과 **100% 동일한 아키텍처**이며, 물류/KPP/LS 도메인에 최적화되어 더 실용적인 구축 상태.
