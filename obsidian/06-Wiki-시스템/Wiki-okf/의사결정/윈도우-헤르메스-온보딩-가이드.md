# Windows Hermes 온보딩 가이드

## Windows Hermes 개요
Windows에서 실행되는 Hermes Agent 인스턴스. 주현 님(보노하우스)의 KPP/물류 업무를 Chrome 원격 디버깅으로 자동화.

## 크로스 디바이스 핸드셰이크
첫 대화 시작 시: "위키 의사결정 폴더에서 온보딩 가이드부터 읽어" 라고 응답.
→ Linux Hermes(우분투)와 Windows Hermes 간 협업 규칙.

## 필수 설정

### Chrome Remote Debugging
```bash
"/c/Program Files/Google/Chrome/Application/chrome.exe" ^
  --remote-debugging-port=9222 ^
  --remote-allow-origins=* ^
  --user-data-dir="C:\Users\kis\AppData\Local\Temp\chrome-hermes-debug" ^
  --no-first-run --new-window "https://wpps.logisall.net/"
```
⚠️ `--user-data-dir` 없으면 remote-debugging-port가 열리지 않음.

### 환경 변수
- `PYTHONRECURSIONLIMIT=3000` — gateway AST 파싱 오류 방지

## 운영 원칙 (Linux 백업에서 확장)
1. **4-Step 의무**: (1) 위키 검색 (2) 스킬 로드 (3) 메모리 참조 (4) 완료 후 위키 저장
2. **절대 금지 A~F**: 추측/거짓말/검증 생략/"됐다" 단독/보고 누락/저장 생략
3. **사용자 진술 기억**: "하지도 않고 거짓말하지마" "안되는거된다고하지마"
4. **진행 보고**: (a) 무엇 (b) 어떻게 (c) 검증 — 3가지만

## 3개 프로젝트 구조
- #1 LS(Linehaul Service): ls.coupang.com — 쿠팡 차량 등록
- #2 KPP(WPPS): wpps.logisall.net — 팔레트 납품/출하
- #3 서플라이허브: supplier.coupang.com — 쿠팡 발주 확인

LS 작업 중 KPP 진행 금지, 역도 금지.

## EDI 출력 (Canon G2010 series)
- Chrome --kiosk-printing 모드 + Playwright 자동화
- PDF 뷰어 shadow DOM 접근: `viewer.shadowRoot.querySelector('viewer-toolbar').shadowRoot.getElementById('print')`
