## 2. stale data 가드 (자동 등록 전 무조건 실행)

**2026-06-19 사용자 명시 정책** (재발 방지 영구 조치):
- **VF ls_data.json이 오늘 날짜가 아니면 자동 등록 금지**
- **자체 판단으로 stale data 등록 절대 금지 — 사용자 지시 대기**

### 3중 검증 절차

| 단계 | 확인 | 도구 | 통과 조건 |
|:----:|:-----|:-----|:---------:|
| 1 | VF DB 파일 수정일 | `stat` (Linux) / `ls -l` (Windows) | `< 24시간` |
| 2 | VF API 응답의 `date` 필드 | `curl /departure/api/ls-data` | `date == today` |
| 3 | record 내부 dates | Python `max(dates)` | `>= today` |

### 한 줄 검증 명령어 (실측 2026-06-19)

```bash
curl -s http://localhost:5176/departure/api/ls-data | python3 -c "
import json, sys, datetime
d = json.load(sys.stdin)['data']
dates = [r['date'] for r in d if 'date' in r]
today = datetime.date.today().strftime('%Y-%m-%d')
if not dates or max(dates) < today:
    print(f'STALE: latest={max(dates) if dates else \"none\"} today={today}')
    sys.exit(1)
print(f'OK: latest={max(dates)}')
"
```

### stale data 감지 시 동작

1. 14시 cron 즉시 중단 (Phase 1: VF 조회에서 exit 1)
2. `circuit_breaker.py fail ls-loop-14h data-freshness "<reason>"` 기록
3. Telegram/Slack 가능 시 → 사용자 알림
4. Telegram 미설정 시 → cron 보고에 stale data 명시 후 silent
5. **자체 판단으로 stale data 등록 절대 금지 — 사용자 지시 대기**

### 실측 사례 (2026-06-19 14:00:53)

- VF ls_data.json = 2026-06-17 20:44:45 (2일 전)
- VF API 응답 `date` 필드 = "2026-06-17" (3건 모두)
- 오늘 = 2026-06-19
- → 14시 cron 중단, circuit_breaker 1/3 fail 기록
- → 사용자에게 stale data 보고 + LS 등록 보류 사실 + 쿠키 만료(10일) 추가 보고
