# 백테스트 v3 엔진 전면 개선 완료 (2026-05-28)

## 배경

기존 v2 백테스트 엔진이 286M% 수익률(럭스피아) 같은 **방법론 오류**를 발생시킴. 3가지 근본 원인:
1. **전액 베팅**: 매번 현금의 95%를 단일 종목에 투입
2. **유동성 무시**: 일평균 거래량 무시하고 대량 주문 가능
3. **슬리피지 고정**: 대량 주문도 0.15% 고정

## 해결: 3중 방어 체계

| 방어 | 설정값 | 효과 |
|------|--------|------|
| 분산투자 | POSITION_RATIO = 0.20 | 현금의 20%만 단일 종목 투입 |
| 유동성 제한 | VOLUME_LIMIT_RATIO = 0.02 | 일평균 거래량의 2%까지만 주문 |
| 동적 슬리피지 | 0.15% ~ 2% | 거래대금 침투율 기반 동적 적용 |

## 성능 최적화

- 누적합(cumsum) 기반 avg_vol_lookup O(1) 조회 → 2,878종목 × 19전략 처리 속도 정상화
