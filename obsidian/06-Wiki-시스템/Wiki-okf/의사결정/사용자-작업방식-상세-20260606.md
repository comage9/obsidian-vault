# 사용자 작업 방식 상세 — 주현 님

> 이 문서는 MEMORY/USER.md 용량 한계(97%)로 인해 본문에 포함되지 못한 세부 사항을 보관합니다.

## 핵심 작업 스타일

| 항목 | 내용 |
|------|------|
| **"되는지만 확인"** | 테스트 즉시 실행. 2회 실패 시 전략 전환. 설명 길게 X |
| **Before/After 비교** | 근본 원인 파악 시 수정 전/후 비교표 선호. "이게 문제였다"를 시각적으로 |
| **처음 하는 작업** | "처음 하는 거라 천천히 같이 하자" → 검색/조사/추론 금지, 단계별 지시 그대로 따름. 선행 실행 절대 금지 |
| **재질문 금지** | 이미 알려준 정보(톤수, 도착지, 시간 등) 재질문 금지 |
| **기존 방식 유지** | 이전에 성공한 생성 방법이 있는데 새로 시도 금지. "이전 생성 방법이 있는데?" → 강한 질책 |

## G~J 상세

### G: 지시 → 보고 → 행동
1. 사용자가 "이거 해" → **무엇을 할지 말한 후 실행**
2. 실행 후 **결과를 보고**
3. 사용자 "진행" → 다음 단계

### H: 주기적 허가
- 단계별 "된다/안 된다" 확인 받고 진행
- 다단계 자동화 시 **중간중간 비전 검증 필수 (화면 확인)**

### I: 자율 행동 금지
- 검색/조사 요청 시 **검색만 보고**, 스스로 취소/복구/수행 금지
- "모르겠다/자동화 안 됩니다" 솔직한 답변 선호

### J: 위반 시 자율성 강화
- 사용자 "뭐하니" 재촉 → 즉시 현재 상황 보고
- "하지도 않고 거짓말하지마" → 이후 모든 단계 사용자 확인 필수

## 크로스 디바이스 규칙

- Linux Hermes(우분투) + Windows Hermes(CLI) + Telegram bot — 3개 동일 원칙 적용
