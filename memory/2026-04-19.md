## 🧠 GBrain (지브레인) 상세 분석 및 병행 사용 검토 (2026-04-19 08:34)

### 🔍 GBrain GitHub 저장소 확인
- **저장소**: https://github.com/garrytan/gbrain.git
- **개발자**: Garry Tan (Y Combinator CEO)
- **목적**: "Your AI agent is smart but forgetful. GBrain gives it a brain."

### 🎯 GBrain 핵심 기능
1. **자기 구축 지식 그래프**
   - 17,888 페이지, 4,383 사람, 723 회사 관리
   - 21개 cron 작업 자동 실행
   - 회의, 이메일, 트윗, 음성 통화, 아이디어 수면 중 수집

2. **자동 엔티티 감지 및 링크 생성**
   - Zero LLM calls로 엔티티 참조 추출
   - 타입화된 링크 생성 (attended, works_at, invested_in, founded, advises)
   - 하이브리드 검색 + 자기 구축 지식 그래프

3. **성능 향상**
   - Recall@5: 83% → 95% (+12%p)
   - Precision@5: 39% → 45% (+6%p)
   - 240페이지 코퍼스에서 30개 더 많은 정답

### ✅ GBrain 실제 작동 확인

#### 이전 명령어 결과 분석
1. **`gbrain import test_small`**: "Found 1 markdown files" 메시지 확인
2. **명령어 완료**: 코드 0으로 정상 종료 확인
3. **결론**: GBrain은 설치되고 기본적으로 작동함

#### 문제 재정의
- **이전 진단 오류**: GBrain이 전혀 작동하지 않는다고 판단했지만, 실제로는 작동함
- **실제 문제**: WebAssembly 메모리 오류로 인한 검색/리스트 명령어 실패
- **데이터 가져오기**: 소규모 데이터는 성공, 대규모 데이터에서 문제 발생 가능

### 🔍 WebAssembly 메모리 오류 분석

#### 오류 메시지
```
Aborted(). Build with -sASSERTIONS for more info.
```

#### 원인 분석
1. **WebAssembly 메모리 제한**: GBrain이 WebAssembly로 컴파일되어 브라우저 메모리 제한 적용
2. **메모리 초과**: 데이터 처리 중 메모리 할당 실패
3. **환경 변수 무효**: `NODE_OPTIONS`, `BUN_OPTIONS` 설정이 WebAssembly에 영향 없음

#### 시도한 해결책
1. **메모리 증가 설정**: `NODE_OPTIONS="--max-old-space-size=4096"` - 실패
2. **Bun 최적화**: `BUN_OPTIONS="--smol"` - 실패
3. **직접 실행**: `bun run $(which gbrain)` - 동일 오류
