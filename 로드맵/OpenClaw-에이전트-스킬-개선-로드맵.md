## 현재 문제점

1. API 키/토큰 만료 감지 없음
2. Rate Limit 대응 없음
3. 스킬(자동화 워크플로우) 테스트 부재
4. 메모리 동기화 불일치

---

## 단기 (1-2개월): 훅과 가이드라인을 활용한 즉각적인 오류 통제

> 하네스 엔지니어링의 기본인 **강력한 제약(가드레일)** 설정

### 1. API 검증 훅(Hook) 구축
- `pre_tool_use` 훅 설정
- 스킬 실행 전 API 키 만료 여부 검증
- Rate Limit 한도 초과 여부 선행 체크
- **산출물**: `hooks/pre_tool_check.py`

### 2. 정책 문서화
- `skill.md` 파일 내 행동 매뉴얼 기록
- "Rate Limit 발생 시 대기 후 재시도"
- "무료 API 엔드포인트로 우회"
- 세션 시작 시마다 반복 숙지

### 3. 어드바이저(Advisor) 아키텍처 도입
- 단순 반복 작업: 저렴 모델 (Sonnet/Haiku/Qwen)
- 복잡한 오류 해결/판단: 큰 모델 (Opus) 호출
- 계층형 아키텍처로 토큰 최적화

---

## 중기 (3-6개월): 교차 검증 및 평가 루프 내재화

> 에이전트의 '자기 평가 함정' 탈출

### 1. QA 전담 서브 에이전트 분리
- 메인 에이전트와 분리된 테스트/디버깅 전담 에이전트
- Gstack 방법론 적용
- 병렬 실행으로 객관적 테스트 보장
- **산출물**: `agents/qa_only/`

### 2. 평가 주도 개선 파이프라인
- 테스트 코드 실행 및 점수 매기기
- Playwright MCP로 웹 UI 자동 검증
- **산출물**: `tests/evaluation/`

### 3. 퇴근 전 책임제
- "모든 검증 체크리스트 달성 전 세션 종료 불가"
- 강제 훅(Hook)으로 구현
- **산출물**: `hooks/pre_session_end.py`

---

## 장기 (6개월+): 지식 그래프 기반 장기 기억과 전사적 오케스트레이션

> 파편화된 세션/메모리 불일치 해결

### 1. 메모리 찌꺼기 청소
- `ACP close` 명령어 활용
- TTL(Time-To-Live) 설정
- 불필요 세션 자동 정리
- cron 작업으로 자동화

### 2. SQLite 기반 장기 기억
- 성공 스킬, API 위치, 과거 오류 로그 실시간 기록
- 검색(Search) 기능 연동
- **산출물**: `memory.db`

### 3. 전역 규칙 레포지토리
- 팀 전체 규칙/스킬 파일 중앙 집중 관리
- Git 서브모듈(Submodule)로 각 프로젝트 연결
- 모든 서브 에이전트가 단일화된 최신 지침 사용
- **산출물**: `global-rules/`
