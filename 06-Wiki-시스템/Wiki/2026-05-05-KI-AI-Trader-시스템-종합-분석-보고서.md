## 3. 손절가 / 익절가 시스템 — 4중 방어선

### 3-1. 1차 방어선: UnifiedRiskConfig (고정 값)

| 전략 | 손절 | 익절 | Trailing 활성 | Trailing 간격 |
|------|------|------|--------------|--------------|
| **단타 (DAY_TRADING)** | **-3%** | **+5%** | 수익 +2.5% 시 | 1.5% |
| **중기 (MEDIUM_TERM)** | **-8%** | **+15%** | 수익 +10% 시 | 5.0% |

---

### 3-2. 2차 방어선: Adaptive Trailing Stop (4조건 가중 평균)

**위치:** `adaptive_trailing_stop.py` — 실시간 모니터링의 핵심

**동적 손절 공식:**
```
동적 손절 = ATR(25%) + RSI(25%) + MACD(25%) + Regime(25%)
```

**ATR 기반 손절:**
| ATR 범위 | 동적 손절 |
|---------|---------|
| 저변동성 (ATR < 1.5%) | -1.5% |
| 중변동성 (1.5~2.5%) | -2.5% |
| 고변동성 (2.5~3.5%) | -3.5% |
| 초고변동성 (3.5%+) | -5.0% |

**시장 Regime 구분 (ADX 기반):**
| Regime | 조건 | 손절 보정 |
|--------|------|---------|
| STRONG_UPTREND | ADX > 25, +DI > -DI | -4.5% (여유) |
| WEAK_UPTREND | ADX ≤ 25, +DI > -DI | -3.5% |
| SIDEWAYS | ADX < 20 | -2.5% |
| WEAK_DOWNTREND | ADX ≤ 25, +DI < -DI | -2.0% |
| STRONG_DOWNTREND | ADX > 25, +DI < -DI | -1.5% (타이트) |

---

### 3-3. 3차 방어선: 긴급 손절 (Dynamic Emergency Stop)

**활성화:** ATR 급등, RSI 과매수, MACD 하락 가속, Regime 하락 중 2개 이상 충족 시

**계산:**
```
긴급 손절 = -5% + RSI调整 + MACD调整 + Regime调整
```

| 조정 요소 | 조건 | 보정값 |
|----------|------|-------|
| RSI | < 35 (과매도) | +1.5% |
| RSI | 35~50 | +0.5% |
| RSI | 50~65 | 0% |
| RSI | 65~75 | -0.5% |
| RSI | > 75 (과매수) | -1.0% |
| MACD 히스토그램 | > 0 | +0.5% |
| MACD 히스토그램 | < -0.1 | -0.5% |
| Regime | 강한상승 | +1.0% |
| Regime | 강한하락 | -1.0% |

**범위:** -7% ~ -2%

---

### 3-4. 4차 방어선: 당일 최대 손한도

**한도:** -7% (어떤 상황에서도 초과 불가)

---

### 3-5. 익절 (Take Profit) 시스템

**고정 익절:**
| 전략 | 익절율 |
|------|-------|
| 단타 | +5% |
| 중기 | +15% |

**동적 익절 조정:**
| 요소 | 조건 | 보정 |
|------|------|------|

| 거래량 배수 | ≥ 2.5x | +2% |

## 4. 트레일링 스탑 시스템

### 4-1. 기본 Trailing Stop (adaptive_trailing_stop.py)

**활성화 조건 (OR):**
1. 수익률 ≥ 2.5%
2. RSI ≥ 75
3. MACD 히스토그램 < -0.05 (하락 가속)
4. 변동성 급증 모드

**Trailing 간격:** 1.5%

**부분 익절 조건:**
| 조건 | 행동 |
|------|------|
| RSI ≥ 72 + MACD 데드크로스 | 50% 부분 익절 |
| RSI ≥ 75 + MACD histogram < -0.1 + MACD < Signal | 50% 부분 익절 |
| STRONG_UPTREND | RSI/MACD 매도 신호 **무시**, trailing만 적용 |

---

### 4-2. Morning Gap v2 Trailing (morning_gap_strategy_v2.py)

**핵심:** 진입 직후 즉시 활성화 (수익 대기 없음)

**간격:**
| 시간대 | Trailing Step |
|--------|-------------|
| 09:30 진입 ~ 10:00 | 1.0% |
| 10:00 이후 | 0.3% (타이트) |
| 10:30 | 강제 청산 |

**동작 방식:**
```
진입 → trailing_stop_price = buy_price × 0.99
최고가 갱신 → trailing_stop_price 상향 (단, 당일 종가 기준만)
10:00 이후 → trailing_step 0.3%로 축소 + 즉시 재계산
```

## 7. 현재 시스템의 문제점 및 개선 필요 사항

### ⚠️ 중복/충돌 문제

| 문제 | 파일 | 설명 |
|------|------|------|
| **중복 Trailing** | `adaptive_trailing_stop.py` + `morning_gap_strategy_v2.py` | 두 파일 모두 trailing 로직 보유 |
| **손절값 불일치** | `unified_risk.py` vs `adaptive_trailing_stop.py` | DAY_TRADING: -3% vs 동적 평균: ~-3.5% |
| **HMM 하드코딩** | `ki_project_monitor.py` | 011200(HMM)만 15% 익절, 나머지 적용 안 됨 |
| **동적 익절 미구현** | `unified_risk.py` | 주석으로는 있다，但实际上 코드에 없음 |

### ⚠️ architecture 문제

| 문제 | 설명 |
|------|------|
| **실시간 실행 경로 불분명** | `adaptive_trailing_stop.py`이 매 수 후 실시간으로 호출되는지가 명확하지 않음 |
| **멀티 에이전트 미연동** | `multi_agent_arena.py`가 실제로 실시간 거래와 연결되는 경로 미확인 |
| **WebSocket vs REST** | 조건검색은 WebSocket, 주문은 REST — 동기화 이슈 가능성 |

### ⚠️ 백테스트 전용 파일

| 파일 | 상태 |
|------|------|
| `condition_strategies.py` | 백테스트 전용 — 실시간 연동 안 됨 |
| `simple_trading_strategy.py` | 백테스트 전용 — 실시간 연동 안 됨 |
| `scripts/stoploss_swing.py` | 백테스트 전용 |

```

```

```
      adaptive_trailing_stop.py로 통합 (별도 유지 시 중복 문제)
```

### Phase 3: 실행 아키텍처 정리

