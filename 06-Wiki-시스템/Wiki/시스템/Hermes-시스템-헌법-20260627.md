> 최종 갱신: 2026-06-27

## §0. Pre-flight Check (작업 시작 시)

| Step | 액션 |
|------|------|

## §2. §7 결정 고정 (Decision Lock) — 최상위

```
사용자가 다음 중 1회 응답 = 확정:
- "네 / 진행 / 진행해 / 네 진행 / OK / 응 / 맞아 / 그래 / 하자"

→ 재질문 절대 금지
→ 확정된 계획 즉시 실행
→ 오류 발생 시에만 중단 + 보고
→ Pre-flight Step 5보다 우선
```

**재질문 시 위반 로그**: `[DECISION LOCK VIOLATION]` 출력 후 즉시 실행.

## §6. 3중 완료 (NOTICE)

모든 작업 완료 = (a) Wiki 기록 (b) README 갱신 (c) Git push

**부분 완료 보고 금지**. 어느 하나라도 빠지면 "완료" 아님.

**Push 시 안전 통합**: 다른 에이전트 작업이 origin에 있을 수 있음.
```
git fetch origin
git rebase --autostash origin/master  # stash + rebase + unstash 자동
git push origin master
```
다른 에이전트의 `unstaged`/`untracked` 변경은 절대 건드리지 않음.
