```
│           HermesTrinityCoordinator                       │
│                                                         │
│  coordinate(query, max_turns=5)                         │
│  ├── Turn 1: Thinker → 전략 수립                        │

│  │                                                     │
│  ├── Turn 2: Worker → 실행                             │
│  │   └── _worker_llm()                                 │
│  │        - 주문 실행 (매수/매도)                        │
│  │        - 분석 결과 산출                              │
│  │                                                     │
│  └── Turn 3+: Verifier → 검증                          │
│       └── _verifier_llm()                               │
│            - ACCEPT / REVISE / REJECT                   │
│            - 리스크 요인 분석                            │
│            - 조정 사항 제안                              │
└─────────────────────────────────────────────────────────┘
```

## 3. 핵심 클래스

### TrinityRole (Enum)
```python
THINKER = "thinker"   # 전략 수립
WORKER = "worker"     # 실행
VERIFIER = "verifier"  # 검증
```

### Verdict (Enum)
```python
ACCEPT = "accept"   # 승인
REVISE = "revise"   # 수정 필요
REJECT = "reject"    # 기각
```

### TrinityMessage
```python
@dataclass
class TrinityMessage:
    turn: int
    role: TrinityRole
    agent_id: str
    content: str
    verdict: Optional[Verdict]
    timestamp: datetime
```

### TradeIntent
```python
@dataclass
class TradeIntent:
    action: str           # "BUY", "SELL", "HOLD"
    stock_code: str
    quantity: int
    price: float
    reason: str
    stop_loss_pct: float = -3.0
    take_profit_pct: float = 5.0
    trailing_active: bool = True
    trailing_step: float = 1.5
```

### RiskAssessment
```python
@dataclass
class RiskAssessment:
    verdict: Verdict
    reason: str
    risk_factors: List[str]
    adjustments: Dict[str, Any]
    confidence: float
```

## 4. 주요 메서드

### `coordinate(query, max_turns=5)`
TRINITY 코디에이션 전체 흐름 실행.
- Thinker → Worker → Verifier 순차 실행
- Verifier가 ACCEPT하면 종료
- REVISE이면 다음 턴에서 Thinker 재호출

### `quick_trade(action, stock_code, quantity, price)`
빠른 거래 실행 (TRINITY 미사용 시).
- Verifier 먼저 체크
- ACCEPT 되어야 실제 실행

## 5. 현재 vs TRINITY 적용 비교

| 항목 | 현재 (run_trader_v2) | TRINITY 적용 시 |
|------|---------------------|----------------|
| 에이전트 선택 | 규칙 기반 | Coordinator head |
| 역할 구분 | 없음 | Thinker/Worker/Verifier |
| 종료 조건 | 없음 | Verifier ACCEPT |
| 다중 턴 | 없음 | 최대 5턴 |
| 리스크 검증 | RiskManager 별도 | Verifier 내부 |
