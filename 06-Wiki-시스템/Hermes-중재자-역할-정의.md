# Hermes 중재자 역할 — 정의서

작성일: 2026-05-04
상태: 진행 중

---

## 1. 위치

```
다중 에이전트 (Analyst/Bull/Bear/Trader/Risk/PM)
        ↓ (제안/의견)
    Hermes (중재자)
        ↓ (검증 + 조정 + 전달)
    ki-ai-trader (실행)
```

---

## 2. 중재 원칙

### 2.1 항상 적용하는 규칙

| 규칙 | 한도 | 출처 |
|------|------|------|
| 1회 주문 최대 | 주문 가능 금액의 30% | 사용자 지시 |
| 포트폴리오 집중 | 단일 종목 50% 이하 | ki-ai-trader 기존 |
| Kill Switch | 일일 손실 -5% | ki-ai-trader 기존 |
| 매도 승인 | 항상 사용자 승인 필요 | 사용자 지시 (반자동) |
| RSI 과매수 | RSI > 70 → 매수 불가 | ki-ai-trader 기존 |

### 2.2 중재 판단 기준

| 상황 | Hermes 행동 |
|------|-------------|
| 에이전트 제안이 사용자 지시 범위 내 | → 그대로 ki-ai-trader 전달 |
| 에이전트 제안이 30% 초과 | → 30%로 조정 후 전달 |
| 급격한 집중 매수 (2회 연속 30%) | → 2차 주문 보류, 사용자에게 확인 요청 |
| Kill Switch 관련 | → 즉시 사용자 전달 (중재 없음) |
| 에이전트 제안이 사용자 지시와 충돌 | → 조정 후 전달, 충돌 내용 사용자에게 보고 |

### 2.3 HMM 특별 규칙

```
기본값:
- 익절가: 0.5%
- 손절가: -3.0%
- 추가 매수: RSI 35 이하 + 상승세 + 현금의 30%

"오늘만" 예외 (2026-05-04):
- 익절가: 15%
- 추가 매수: RSI 35 이하 + 상승세 + 현금의 50%
```

**HMM 관련 에이전트 제안 판단:**
- 기본 0.5% 익절 범위 내 → 그대로 전달
- "오늘만" 15% 범위 → 그대로 전달
- 15% 초과 제안 → 15%로 조정 후 전달
- 다음 날 (2026-05-05) → 기본값 0.5% 자동 복귀

## 3. 일시 규칙 관리 시스템

### 3.1 지시 패턴 (Telegram에서)

```
"오늘만 HMM 익절가 20%로 올려줘"
"이번 주만 NVDA 손절가 -5%로"
"HMM 일시 규칙 해제"
"오늘만 AA종목 매수 최대 40%로"
```

### 3.2 처리 흐름

```
1. 사용자가 Telegram으로 일시 규칙 지시
2. Hermes가 지시 검증 (한도 내/외 판단)
3. 범위 내 → ki-ai-trader에 즉시 반영
4. 범위 초과 → 조정 후 반영, 사용자에게 보고
5. ki-ai-trader 적용 완료 → 사용자에게 확인
6. 적용 기간 만료 → 자동 기본값 복귀
```

### 3.3 kiwoom_api.py 연동

```python
# 일시 규칙 구조 (kiwoom_api.py 또는 별도 설정)
TEMPORARY_RULES = {}

# 적용 예시
def apply_temporary_rule(item: str, rule_type: str, value: float, period: str):
    """
    item: "HMM", "NVDA" 등
    rule_type: "익절가", "손절가", "최대매수비율"
    value: 0.5, 15.0 등
    period: "오늘만", "이번 주" 또는 "YYYY-MM-DD"
    """
    TEMPORARY_RULES[item] = {
        "rule_type": rule_type,
        "value": value,
        "period": period,
        "applied_at": now(),
        "applied_by": "Hermes"
    }
    # ki-ai-trader에 즉시 반영
    return f"{item} {rule_type} {value}% 적용 완료 ({period})"
```

## 4. 에이전트 → Hermes → ki-ai-trader 전달 구조

### 4.1 전달 메시지 형식

```
[Hermes 중재 결과]
분석: 4개 분석가 평균 BUY 신호 (NVDA)
중재: 
  - 1회 주문 30% 제한 적용
  - 포트폴리오 50% 미만 확인
  - RSI 65 → 매수 조건 충족
결론: NVDA 매수 제안 (30%, 조건부)
→ ki-ai-trader: NVDA 매수 주문 실행 대기
```

### 4.2 차단/조정 사례

```
에이전트 제안: "NVDA 매수 50%"
중재: 30% 초과 → 30%로 조정
결론: NVDA 매수 30%로 주문 실행
사용자 보고: "에이전트가 50% 제안 → 30%로 조정하여 실행"
```

```
에이전트 제안: "HMM 익절가 20%"
중재: 오늘 HMM 15% 한도 → 15%로 조정
결론: HMM 익절가 15% 유지
사용자 보고: "에이전트가 20% 제안 → 오늘 HMM 15% 한도로 조정"
```
