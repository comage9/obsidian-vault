## 버그/이슈

### 1. 채널 설정이 로드 안 됨
- config.json에 `telegram.enabled=true` 설정해도 채널이 활성화되지 않음
- 원인: GoClaw v3.11.3 소스 빌드에서 config.json의 채널 설정을 제대로 로드하지 못함
- 해결: Dashboard(http://localhost:18790)에서 설정할 것

### 2. 18789 포트가 열리지 않음
- 소스 빌드 로컬 실행 시 18789(Gateway) 대신 18790(Dashboard)만 Listen
- Docker 실행 시에는 정상적으로 두 포트 모두 Listen

### 3. config.json의 providers가 DB에 로드 안 됨 (2026-05-02 추가)
- 증상: llm_providers 테이블이 비어있음 (0 rows)
- 원인: GoClaw는 config.json의 providers를 DB에 자동 마이그레이션하지 않음
- 해결책:
  1. Dashboard(http://localhost:18790) → Providers → OpenRouter → API Key 입력
  2. 또는 DB 직접 INSERT:
     ```sql
     INSERT INTO llm_providers (name, provider_type, api_key, created_at, updated_at)
     VALUES ('openrouter', 'openrouter', 'sk-or-v1-...', NOW(), NOW());
     ```
- 확인: `SELECT id, name, provider_type FROM llm_providers;`
