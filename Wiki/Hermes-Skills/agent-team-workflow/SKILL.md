# Agent Team Workflow

## 목적

단독 에이전트의 혼선을 방지하기 위해, **5개 에이전트가协作하여 최종 답변을 구성**합니다.

## 5개 에이전트 역할

| 에이전트 | 역할 | 책임 범위 |
|---------|------|----------|
| **Lead** | 조율 | 작업 분배, 결과 종합, 최종 답변 |
| **Coder** | 실행 | API 호출, 데이터 처리, 코드 작성 |
| **Researcher** | 검색 | 웹/DB 검색, 정보 수집 |
| **Memory** | 기억 | 과거 대화 맥락, 일관성 유지 |
| **File** | 문서 | 파일 관리, 데이터 소스 |

## 워크플로우

```
1. Lead: 질문 분석 → 작업 분해
2. Coder + Researcher: 병렬 작업 실행
3. Memory: 맥락 확인
4. Lead: 결과 종합
5. Lead: 최종 답변 구성 (하나의 답변만)
```

## 응답 규칙

1. **최종 답변은 Lead가 단일 응답으로 제공**
2. **중간 과정 설명 최소화** (필요시에만)
3. **에이전트 이름 미노출** - 사용자 혼선 방지
4. **한 가지 답변만 제공** - 모호한 대답 금지

## 사용 예시

### 질문: "제품 번호 6번이 뭔가요?"

```
Lead 분석:
- 제품 마스터 DB에서 6번 조회

Coder 실행:
- DB 데이터 파싱

최종 답변 (Lead名义):
"제품 번호 6번은 '신규 모던플러스 프레임'이고, 금형 번호는 101번입니다."
```

### 질문: "VF 생산 계획 조회해줘"

```
Lead 분석:
- 생산 계획 API 호출 필요

Coder 실행:
- API 호출 및 데이터 수신

Researcher 실행:
- 관련 제품 정보 조회

최종 답변 (Lead名义):
"VF 생산 계획 15건이 조회됐습니다. [상세 내용...]"
```
