## 2. 핵심 수정 및 보완 사양 설명

### ① 순차적 인쇄 대기(4초 딜레이) 구현
* **현상**: Adobe Acrobat에서 `"파일 이름, 디렉터리 이름 또는 볼륨 레이블 구문이 잘못되었습니다"` 에러 팝업이 발생하는 원인은 인쇄 명령이 동시에 중복으로 겹쳐서 OS 인쇄 큐가 충돌을 일으킨 탓입니다.
* **보완**: `os.startfile` 호출 직후 **`time.sleep(4)`**를 부여하여, 이전 인쇄 스풀이 윈도우 스풀러에 온전히 수신 완료된 후 다음 차량 인쇄가 들어가도록 안전장치를 적용했습니다. 또한 경로 충돌을 방지하기 위해 윈도우 OS 표준 경로로 완벽히 정규화(`os.path.normpath`)하여 전달합니다.

### ② `--print` 파라미터 제어 옵션 도입
* 오늘 등록된 전체 차량 중 **원하는 차량만 골라서 인쇄**하거나, 아예 인쇄를 스킵할 수 있는 매개변수(`--print`) 필터링 기능을 탑재하였습니다.

## 3. 명령어 사용 방법 및 예제

터미널(PowerShell 또는 CMD)에서 다음 명령어로 스크립트를 조작할 수 있습니다.

### 예제 1: 1~3호차 전체를 중복 없이 순차 인쇄 (기본값)
1호차 ➔ 4초 대기 ➔ 2호차 ➔ 4초 대기 ➔ 3호차 순으로 순차적으로 딱 1장씩 안정적으로 인쇄합니다.
```bash
python E:\coding\skill\KPP\wpps_register_2_3.py --print all
```

### 예제 2: 2호차만 타겟팅하여 1장 인쇄
1호차와 3호차는 등록만 완료하고 인쇄는 건너뛰며, 오직 **2호차**에 대해서만 1장 출력을 실행합니다.
```bash
python E:\coding\skill\KPP\wpps_register_2_3.py --print 2
```

### 예제 3: 1호차와 3호차만 선택하여 인쇄
쉼표(`,`) 구분자를 이용해 **1호차**와 **3호차**만 선택적으로 출력합니다.
```bash
python E:\coding\skill\KPP\wpps_register_2_3.py --print 1,3
```

### 예제 4: 인쇄 없이 출하통보등록만 진행
모든 인쇄 동작을 스킵하고 오직 KPP 시스템의 출하통보등록 및 일괄 저장까지만 깨끗하게 완료합니다.
```bash
python E:\coding\skill\KPP\wpps_register_2_3.py --print none
```

### 예제 5: KPP에서 특정 호차의 EDI 전표만 단독 자동 인쇄
이미 출하통보등록이 완료된 상태에서, **KPP EDI 전표 팝업을 띄우고 백그라운드에서 자동으로 Enter 키를 입력**하여 인쇄까지 100% 무인 완료합니다. (예: 2호차만 단독 출력)
```bash
python E:\coding\skill\KPP\wpps_register_2_3.py --kpp-print 2
```

## 3.1 프린터 자동 확인 및 기본 프린터 변경 기능

본 자동화 프로그램은 실행 즉시 Windows 시스템의 기본 프린터 설정을 강제로 검사하고 교정합니다.

* **동작 원리**: 
  - 스크립트 가동 즉시 OS 기본 프린터를 조회하여 `Canon G2010 series`로 지정되어 있는지 확인합니다.
  - 만약 다른 프린터(예: 사무용 대형 복합기 등)가 기본으로 지정되어 있을 경우, **자동으로 `Canon G2010 series`로 기본 프린터를 변경**하여 스풀러 오작동을 방지합니다.
  - 이 조치를 통해 **LS PDF 인쇄** 및 **KPP 크롬 인쇄 다이얼로그의 기본 인쇄 대상(Destination) 설정**이 항상 안전하게 `Canon G2010 series` 프린터로 자동 포커싱됩니다.

## 4. 트러블슈팅 요약

1. **Acrobat 에러 팝업 재발 시**:
   - 여러 창이 이미 켜져 있는 프로세스 충돌일 수 있으므로 작업 관리자에서 `Acrobat` 프로세스를 모두 종료하고 실행 간 대기 시간을 더 늘리십시오.
2. **프린터 반응 없음**:
   - `os.startfile`은 Windows의 **기본 프린터(Default Printer)**로 즉시 큐를 전송합니다. 제어판 ➔ '장치 및 프린터'에서 Canon G2010 series가 기본 프린터로 체크되어 있는지 확인하십시오.
