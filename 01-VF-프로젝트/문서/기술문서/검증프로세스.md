# 검증 프로세스

## 5단계 검증 흐름 (반드시 순서대로)
1. **코드 수정 완료**
2. `npm run build` — 빌드 확인
3. `curl` — 서버 응답 확인
4. **Claude Code로 실제 검증** (playwright/cli 등)
5. **검증 통과 후 Git push**

## 검증 도구 사용 규칙
- **playwright:** 페이지 렌더링/로딩 검증
- **Claude Code:** 코드 검증 + 동작 테스트
- "검증해라/테스트해라" 요청 → **실제로 도구 호출** (말만 하지 않기)

## 금지 사항
- 빌드 성공 ≠ 동작 확인
- "검증할게요" → 반드시 실제 검증 도구 사용
- 수정 → 빌드 → push → 끝 금지

## CIVC 검증 (Verify)
- PreCompletionChecklist: 작업 완료 전 자동 체크
- 같은 파일 3회 이상 수정 시 접근 방식 재고 (둠 루프 방지)
